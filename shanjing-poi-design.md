## 1. 概述

### 1.1 设计目标
本文档定义山径APP在轻度徒步场景下的POI（Point of Interest，兴趣点）设计与表达规范，旨在：
- **降低迷路焦虑**：通过清晰的导航类POI，让用户随时知道自己在哪、该往哪走
- **提升安全感**：通过安全类POI的强化设计，让女性用户感到安心
- **优化体验流畅度**：通过服务类POI的合理布局，让补给和休息无压力
- **增加探索乐趣**：通过信息类和社交类POI，丰富徒步的趣味性

### 1.2 目标用户特征
| 维度 | 描述 |
|------|------|
| **人群** | 25-32岁城市白领，女性为主（60%+） |
| **场景** | 周末1天往返，5-10km轻度徒步 |
| **团队** | 1-3人小团体，或独自出行 |
| **痛点排序** | 怕迷路 > 担心安全 > 找不到补给 |
| **行为特征** | 手机依赖度高、安全意识强、审美要求高、不愿学习复杂操作 |

### 1.3 设计原则
1. **安全第一**：安全类POI必须最醒目、最易获取
2. **简洁直观**：图标和交互设计符合直觉，无需学习成本
3. **场景适配**：户外强光环境下依然清晰可见
4. **渐进披露**：基础信息一眼可见，详细信息点击展开
5. **情感化设计**：用温暖、鼓励的文案缓解用户焦虑

## 2. POI 分类体系

### 2.1 分类框架

| 类型 | 优先级 | 设计权重 | 核心作用 |
|------|--------|----------|----------|
| **安全类** | P0 | 最高 | 保障用户生命安全 |
| **导航类** | P0 | 高 | 确保用户不迷路 |
| **服务类** | P1 | 中 | 提升徒步舒适度 |
| **信息类** | P2 | 低 | 增强徒步趣味性 |
| **社交类** | P2 | 低 | 满足分享需求 |

### 2.2 安全类 POI（P0）

**设计原则**：最醒目、最优先、永不遮挡

| POI类型 | 图标样式 | 颜色 | 使用场景 | 特殊设计 |
|---------|----------|------|----------|----------|
| **紧急避险点** | 🏠 房屋+盾牌 | 安全红 #E53935 | 恶劣天气、身体不适时的避难所 | 点击一键导航，显示容纳人数、是否有信号 |
| **信号覆盖区** | 📶 信号塔 | 信号绿 #43A047 | 需要联系外界时的通信点 | 显示运营商信号强度（移动/联通/电信） |
| **下撤点** | ↘️ 下箭头+路 | 警示橙 #FB8C00 | 无法完成全程时的安全退出点 | 显示下撤后到达的村庄/公路位置 |
| **医疗点** | ➕ 十字 | 医疗红 #D32F2F | 受伤或突发疾病时的救助点 | 显示距离、是否有值班人员 |
| **SOS求救点** | 🆘 SOS标识 | 紧急红 #B71C1C | 配备救援设备的固定点位 | 长按3秒触发SOS，自动发送位置 |

**安全类POI强化机制**：
- **视觉强化**：安全类POI图标比其他类型大20%，始终置顶显示
- **SOS快捷入口**：导航页面常驻SOS按钮，点击显示最近的安全POI
- **预警推送**：接近关键安全POI时，提前100米语音提醒
- **离线优先**：安全POI信息必须包含在离线包中，确保无网络时可用

### 2.3 导航类 POI（P0）

**设计原则**：清晰指示方向，消除选择困惑

| POI类型 | 图标样式 | 颜色 | 使用场景 | 交互设计 |
|---------|----------|------|----------|----------|
| **起点** | 🚩 旗帜 | 起点绿 #4CAF50 | 徒步开始位置 | 显示集合建议、停车信息 |
| **终点** | 🏁 终点旗 | 终点蓝 #2196F3 | 徒步结束位置 | 显示解散建议、交通接驳 |
| **关键路口** | 🔀 分叉路 | 导航紫 #7E57C2 | 容易走错的路口 | 实景照片对比+箭头指引 |
| **观景点** | 👁️ 眼睛 | 景观青 #00897B | 视野开阔的观赏点 | 显示最佳观赏时间、拍照建议 |
| **海拔标记** | ⛰️ 山峰+数字 | 山地棕 #795548 | 重要海拔节点 | 显示累计爬升/下降 |
| **距离标记** | 📍 标记+km | 里程灰 #757575 | 每1-2公里的标记点 | 显示已走/剩余距离 |

**导航类POI特殊设计**：
- **路口指引强化**：关键路口必须配备实景照片，采用"照片对比"模式
- **语音播报**：距离关键路口50米时自动语音提醒"前方30米右转"
- **方向箭头**：在地图上用3D箭头指示行进方向，随用户朝向旋转
- **进度感知**：在起点、中点、终点显示进度环，增强成就感

### 2.4 服务类 POI（P1）

**设计原则**：实用为主，不干扰核心导航

| POI类型 | 图标样式 | 颜色 | 使用场景 | 信息展示 |
|---------|----------|------|----------|----------|
| **卫生间** | 🚻 卫生间 | 服务蓝 #1976D2 | 公共厕所或自然遮蔽点 | 显示类型（旱厕/水厕）、卫生状况 |
| **补给点** | 🥤 水杯+食物 | 补给橙 #F57C00 | 可购买水/食物的地方 | 显示营业时间、商品类型、价格区间 |
| **休息区** | 🪑 长椅 | 休息绿 #388E3C | 有座椅的休息点 | 显示容纳人数、是否有遮阳/避雨 |
| **垃圾桶** | 🗑️ 垃圾桶 | 环保绿 #2E7D32 | 垃圾分类回收点 | 提示"无痕山野"理念 |
| **水源点** | 💧 水滴 | 水源蓝 #0277BD | 山泉水或饮用水点 | 显示水质说明（可饮用/需煮沸） |
| **停车场** | 🅿️ P字 | 停车灰 #616161 | 自驾停车位置 | 显示车位数量、收费标准 |

**服务类POI布局策略**：
- **密度控制**：每2-3公里至少1个服务类POI，避免信息过载
- **智能筛选**：根据用户行进方向，只显示前方1公里内的服务POI
- **营业时间提醒**：接近补给点时，若已关门则显示灰色图标并提示
- **用户反馈**：允许用户上报服务POI状态（如"卫生间维修中"）

### 2.5 信息类 POI（P2）

**设计原则**：丰富体验，不造成信息负担

| POI类型 | 图标样式 | 颜色 | 使用场景 | 内容形式 |
|---------|----------|------|----------|----------|
| **景点介绍** | 📖 书本 | 文化棕 #6D4C41 | 自然景观/历史遗迹介绍 | 图文卡片，支持语音播报 |
| **植物标识** | 🌿 叶子 | 自然绿 #558B2F | 特色植物科普点 | 植物照片+名称+简介 |
| **地质奇观** | 🪨 岩石 | 地质灰 #455A64 | 特殊地质景观 | 地质形成原理图解 |
| **历史遗迹** | 🏛️ 古建筑 | 历史黄 #F9A825 | 人文历史点位 | 历史背景+故事讲解 |
| **最佳拍照点** | 📷 相机 | 摄影紫 #6A1B9A | 推荐摄影位置 | 示例照片+拍摄参数建议 |

**信息类POI展示策略**：
- **被动触发**：默认不显示，用户主动探索时展开
- **语音讲解**：支持TTS语音播报，解放双眼
- **离线内容**：景点介绍文字预下载，图片按需加载
- **轻量设计**：信息卡片控制在3屏以内，避免阅读疲劳

### 2.6 社交类 POI（P2）

**设计原则**：满足分享欲，但不强制社交

| POI类型 | 图标样式 | 颜色 | 使用场景 | 功能设计 |
|---------|----------|------|----------|----------|
| **打卡点** | 📍 定位+星 | 打卡粉 #C2185B | 推荐打卡位置 | 一键生成打卡卡片 |
| **最佳观景点** | 🌅 日出 | 景观橙 #EF6C00 | 视野最佳的观赏点 | 显示当前人数、拥挤度 |
| **拍照打卡框** | 🖼️ 相框 | 创意紫 #7B1FA2 | 有创意装置的拍照点 | AR相框叠加功能（M2） |
| **留言墙** | 💬 对话框 | 社交蓝 #1565C0 | 虚拟留言位置 | 查看他人留言、留下足迹 |

**社交类POI隐私设计**：
- **位置模糊**：社交POI显示时，用户精确位置模糊处理
- **可选参与**：用户可选择不显示在他人社交视图中
- **内容审核**：UGC内容需审核后展示，防止不当信息

## 3. POI 视觉设计规范

### 3.1 图标设计原则

#### 3.1.1 设计标准
| 维度 | 规范 | 说明 |
|------|------|------|
| **风格** | 线性图标+面性填充 | 简洁现代，易于识别 |
| **线条粗细** | 2px | 保证户外强光下清晰可见 |
| **圆角** | 2px | 柔和友好，减少攻击性 |
| **尺寸** | 标准24×24dp，安全类28×28dp | 安全类图标更大更醒目 |
| **颜色** | 分类色+白色背景 | 高对比度，快速识别 |

#### 3.1.2 户外适配设计
- **高对比度**：所有图标在白色背景和地图背景上都清晰可见
- **阴影处理**：图标底部添加1px投影，增强立体感
- **选中状态**：选中时图标放大1.2倍+外发光效果
- **夜间模式**：自动切换为深色模式，降低亮度保护视力

## 4. POI 信息层级与交互设计

### 4.1 信息层级架构

#### 4.1.1 三级信息架构
```
┌─────────────────────────────────────────────────────────────┐
│  L1: 地图图标层（一眼可见）                                    │
│  ├── 图标（类型识别）                                         │
│  ├── 名称（简短，2-4字）                                      │
│  └── 距离（如"500m"）                                        │
├─────────────────────────────────────────────────────────────┤
│  L2: 卡片预览层（点击展开）                                    │
│  ├── 名称 + 类型标签                                          │
│  ├── 距离 + 预计到达时间                                       │
│  ├── 关键信息（如"有信号"、"营业中"）                          │
│  └── 操作按钮（导航/收藏/分享）                                │
├─────────────────────────────────────────────────────────────┤
│  L3: 详情页面层（深入了解）                                    │
│  ├── 实景照片                                                 │
│  ├── 完整描述                                                 │
│  ├── 详细信息（营业时间、容纳人数等）                           │
│  ├── 用户评价/状态                                            │
│  └── 相关POI推荐                                              │
└─────────────────────────────────────────────────────────────┘
```

#### 4.1.2 信息优先级排序
| 优先级 | 信息项 | 展示层级 | 说明 |
|--------|--------|----------|------|
| P0 | POI名称 | L1 | 必须一眼可见 |
| P0 | 距离 | L1 | 帮助用户判断远近 |
| P0 | 类型标识 | L1 | 图标颜色区分 |
| P1 | 预计到达时间 | L2 | 基于当前速度计算 |
| P1 | 关键状态 | L2 | 如"营业中"、"有信号" |
| P2 | 实景照片 | L3 | 帮助确认位置 |
| P2 | 详细描述 | L3 | 补充信息 |
| P3 | 用户评价 | L3 | 社区反馈 |

### 4.2 交互设计

#### 4.2.1 地图图标交互
| 操作 | 响应 | 说明 |
|------|------|------|

| **长按** | 快速导航 | 直接开始导航至该POI |
| **双击** | 放大该区域 | 方便查看周边POI |
| **滑动** | 切换相邻POI | 左右滑动浏览同类型POI |

#### 4.2.3 语音播报设计
| 触发条件 | 播报内容 | 优先级 |
|----------|----------|--------|
| 距离关键路口50m | "前方50米路口右转，进入小路" | P0 |
| 距离安全POI 100m | "前方100米有紧急避险点" | P0 |
| 距离服务POI 200m | "前方200米有补给点" | P1 |
| 到达POI | "已到达观景点，可以休息一下" | P1 |
| 偏航检测 | "您已偏离路线，请返回" | P0 |
| 定时提醒 | "您已徒步1小时，注意休息" | P2 |

**语音播报原则**：
- **简洁**：每条播报不超过15秒
- **及时**：提前量足够用户反应
- **不打扰**：连续播报间隔不少于30秒
- **可关闭**：用户可选择关闭非安全类播报

### 4.3 预警机制设计

#### 4.3.1 接近预警
```
触发条件: 用户距离关键POI一定距离
预警方式: 
  - 视觉: 图标放大+脉冲动画
  - 声音: 语音播报
  - 震动: 短震动（可选）

预警距离配置:
  - 安全类POI: 100米预警
  - 导航类POI: 50米预警
  - 服务类POI: 200米预警
```

#### 4.3.2 错过预警
```
触发条件: 用户经过POI但未停留
预警方式:
  - 视觉: 图标变为灰色+"已错过"标签
  - 声音: "您已错过补给点，下一个在2公里后"
  - 操作: 提供"返回"或"继续"选项

适用场景:
  - 服务类POI（卫生间、补给点）
  - 关键路口（防止走错）
```

#### 4.3.3 危险预警
```
触发条件: 
  - 用户偏离路线>30米
  - 用户进入危险区域
  - 天气突变

预警方式:
  - 视觉: 全屏红色警告+闪烁
  - 声音: 急促提示音+语音
  - 震动: 连续震动
  - 自动: 显示最近安全POI

示例文案:
  "警告：您已偏离路线，最近的下撤点在前方800米"
```

## 5. 轻度徒步特殊需求设计

### 5.1 安全POI强化机制

#### 5.1.1 SOS一键求救
```
触发方式:
  - 导航页面常驻SOS按钮（右下角）
  - 长按3秒触发（防误触）
  - 支持音量键快捷触发（连按5次）

求救流程:
  1. 用户触发SOS
  2. 显示最近的安全POI列表
  3. 自动发送位置给紧急联系人（短信+推送）
  4. 显示最近安全POI的导航
  5. 提供"已安全"确认按钮

发送内容:
  "【山径SOS】我是XXX，在XXX路线遇到紧急情况，
   当前位置：纬度XX.XX，经度XX.XX，
   最近安全点：XX避险点，距离XXX米"
```

#### 5.1.2 安全POI快速导航
```
入口:
  - SOS按钮点击后的推荐列表
  - 导航页面"安全点"快捷入口
  - 语音指令"最近的卫生间在哪"

显示信息:
  - 安全POI名称和类型
  - 当前距离和预计到达时间
  - 该POI的关键信息（如有信号、可容纳人数）
  - 一键导航按钮
```

#### 5.1.3 定时安全提醒
```
提醒频率:
  - 默认每30分钟提醒一次
  - 可自定义间隔（15/30/60分钟）
  - 夜间模式（18:00-06:00）自动延长间隔

提醒内容:
  - "您已徒步30分钟，当前位置正常，注意休息"
  - 显示最近的安全POI
  - 提供"我很好"快速反馈

异常检测:
  - 连续2次未确认提醒，自动发送位置给紧急联系人
  - 长时间静止（>1小时）自动提醒
```

### 5.2 体力评估与POI间距

#### 5.2.1 路线难度评估模型
```
难度等级 = f(总距离, 累计爬升, POI间距, 下撤点数量)

难度分级:
  - 休闲级: 3-5km，爬升<200m，POI间距<1km
  - 轻量级: 5-10km，爬升200-500m，POI间距1-2km
  - 进阶级: 10-15km，爬升500-800m，POI间距2-3km
  - 挑战级: >15km，爬升>800m，POI间距>3km

POI间距建议（轻度徒步）:
  - 卫生间: 每2-3公里
  - 补给点: 每3-5公里
  - 休息区: 每2-4公里
  - 安全点: 每2-3公里（下撤点）
```

#### 5.2.2 体力预警
```
触发条件:
  - 用户速度持续低于路线平均速度的50%
  - 用户在休息区停留时间过长（>30分钟）
  - 距离下一个服务POI>3公里且用户速度下降

预警方式:
  - "您的速度较慢，最近的下撤点在前方X公里"
  - 显示剩余体力评估（基于已走距离和速度）
  - 建议"休息"或"下撤"
```

### 5.3 时间预估与ETA

#### 5.3.1 基于POI的ETA计算
```
ETA计算模型:
  ETA = 当前时间 + Σ(POI间距i / 预估速度i)

预估速度动态调整:
  - 平路: 4-5 km/h
  - 缓坡: 3-4 km/h
  - 陡坡: 2-3 km/h
  - 根据用户实际速度实时校准

ETA显示位置:
  - 导航页面顶部: 到达终点ETA
  - POI卡片: 到达该POI的ETA
  - 进度条: 每个POI的预计到达时间
```

#### 5.3.2 日落预警
```
触发条件:
  - 根据ETA计算，到达终点时间接近日落时间
  - 剩余徒步时间 > 距离日落时间

预警方式:
  - "注意：预计到达时间18:30，日落时间18:45，建议加快步伐"
  - 显示最近下撤点，提供下撤建议
  - 自动开启省电模式（降低屏幕亮度）
```

### 5.4 下撤方案设计

#### 5.4.1 关键POI下撤路线
```
每个关键POI配置:
  - 主路线: 继续前行的方向
  - 下撤路线: 安全退出的方向

下撤点配置:
  - 位置: GPS坐标
  - 下撤后到达: 村庄/公路/停车场名称
  - 下撤距离: 到安全点的距离
  - 下撤难度: 与主路线对比
  - 交通接驳: 是否有公交/打车点

触发场景:
  - 用户主动选择"下撤"
  - 体力不支预警
  - 天气恶化
  - 受伤/生病
```

#### 5.4.2 下撤决策辅助
```
决策信息展示:
  ┌────────────────────────────────────┐
  │  下撤方案对比                       │
  ├────────────────────────────────────┤
  │  方案A: 继续前行                    │
  │  - 剩余距离: 3.5km                 │
  │  - 预计时间: 1小时30分钟            │
  │  - 难度: 中等（有陡坡）              │
  ├────────────────────────────────────┤
  │  方案B: 从XX点下撤                  │
  │  - 下撤距离: 1.2km                 │
  │  - 预计时间: 30分钟                 │
  │  - 到达: XX村（有公交站）            │
  ├────────────────────────────────────┤
  │  [继续路线]  [选择下撤]             │
  └────────────────────────────────────┘
```

## 6. 数据采集标准

### 6.1 POI数据字段定义

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|

| name | String | 是 | POI名称，2-6个汉字 |
| type | Enum | 是 | 分类：安全/导航/服务/信息/社交 |
| subtype | Enum | 是 | 子类型，如"卫生间"、"补给点" |
| latitude | Float | 是 | 纬度，WGS84坐标系 |
| longitude | Float | 是 | 经度，WGS84坐标系 |
| altitude | Float | 否 | 海拔高度（米） |
| priority | Int | 是 | 优先级：0-10，数字越大越重要 |
| status | Enum | 是 | 状态：正常/维护中/已关闭 |

#### 6.1.2 扩展字段（根据类型选填）
| 字段名 | 类型 | 适用类型 | 说明 |
|--------|------|----------|------|
| description | String | 全部 | 详细描述，不超过200字 |
| photos | Array | 全部 | 实景照片URL列表，最多5张 |
| phone | String | 服务类 | 联系电话 |
| business_hours | String | 服务类 | 营业时间，如"09:00-18:00" |
| capacity | Int | 安全/服务 | 容纳人数 |
| has_signal | Bool | 安全类 | 是否有手机信号 |
| signal_carrier | Array | 安全类 | 有信号的运营商 |
| evacuation_to | String | 安全类 | 下撤后到达的地点 |
| evacuation_distance | Float | 安全类 | 下撤距离（公里） |
| audio_guide | String | 信息类 | 语音讲解URL |
| best_time | String | 社交类 | 最佳观赏/拍摄时间 |
| user_rating | Float | 服务类 | 用户评分，1-5分 |

### 6.2 照片采集规范

#### 6.2.1 照片类型与要求
| 照片类型 | 数量 | 拍摄要求 | 用途 |
|----------|------|----------|------|
| **全景照** | 1-2张 | 广角拍摄POI周边环境 | 帮助用户定位 |
| **特征照** | 1-2张 | 特写POI标志性特征 | 确认到达 |

#### 6.2.2 拍摄标准
```
技术要求:
  - 分辨率: ≥1920×1080
  - 格式: JPG
  - 大小: 单张<2MB
  - 方向: 包含GPS方向信息

拍摄规范:
  - 光线: 白天拍摄，避免逆光
  - 角度: 水平拍摄，与人眼视角一致
  - 清晰度: 主体清晰，无模糊
  - 无遮挡: 避免树枝、电线等遮挡

命名规范:
  - 格式: {poi_id}_{序号}_{类型}.jpg
  - 示例: hz_xihu_001_01_panorama.jpg
```

## 8. 技术实现建议

### 8.1 数据存储
```
POI数据存储方案:
  - 服务端: PostgreSQL + PostGIS（空间索引）
  - 离线包: SQLite（轻量级，嵌入APP）
  - 缓存: Redis（热点POI缓存）

数据同步策略:
  - 路线下载时同步POI数据
  - 增量更新：只更新变化的POI
  - 离线优先：安全POI必须包含在离线包
```

### 8.2 地图渲染
```
POI渲染层级:
  - 层级1: 安全类POI（最高，永不遮挡）
  - 层级2: 导航类POI
  - 层级3: 服务类POI
  - 层级4: 信息类/社交类POI

聚合策略:
  - 缩放级别<14时，同类型POI聚合显示
  - 显示聚合数量，点击展开
  - 安全类POI不参与聚合

性能优化:
  - 视口外POI不渲染
  - 距离>5km的POI简化显示
  - 图标使用矢量格式（SVG）
```

### 8.3 语音播报
```
TTS方案:
  - 优先使用系统TTS（节省包体积）
  - 关键POI预录音频（更自然）
  - 支持离线TTS（下载语音包）

播报队列管理:
  - 安全类播报优先插入队列
  - 连续播报间隔≥30秒
  - 用户操作后暂停播报3秒
```

## 9. 总结

### 9.1 核心设计要点
1. **安全第一**：安全类POI必须最醒目、最易获取
2. **简洁直观**：图标和交互符合直觉，无需学习成本
3. **渐进披露**：基础信息一眼可见，详细信息点击展开
4. **场景适配**：户外强光、弱网环境下依然可用
5. **情感化设计**：温暖友好的文案，缓解用户焦虑

### 9.2 关键成功指标
| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| POI准确率 | >95% | 用户反馈+实地核验 |
| 导航准确率 | >98% | 偏航率统计 |
| 用户满意度 | >4.5分 | 评分系统 |
| SOS响应时间 | <10秒 | 技术监控 |
| 离线可用率 | 100% | 功能测试 |
