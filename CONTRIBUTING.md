# 贡献指南

感谢你对山径项目的关注。

## 如何贡献

### 报告问题

1. 检查是否已有相关 Issue
2. 提供复现步骤、环境信息、截图
3. 使用清晰的标题描述问题

### 提交代码

1. Fork 仓库
2. 创建功能分支：`git checkout -b feature/your-feature`
3. 提交更改：`git commit -m "feat: 描述"`
4. 推送分支：`git push origin feature/your-feature`
5. 创建 Pull Request

### 代码规范

- 遵循 Flutter 官方代码风格
- 使用 `flutter analyze` 检查代码
- 保持组件单一职责
- 添加必要的注释

### 提交信息格式

```
feat: 新功能
fix: 修复问题
docs: 文档更新
style: 代码格式（不影响功能）
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```

## 开发流程

1. 从 `main` 分支创建功能分支
2. 开发完成后本地测试
3. 提交 PR，等待 Review
4. 合并后删除分支

## 联系方式

如有疑问，请通过 Issue 讨论。
