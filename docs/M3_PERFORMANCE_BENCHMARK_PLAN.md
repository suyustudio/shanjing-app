# M3 性能基准测试计划

**版本**: v1.0.0+3  
**日期**: 2026-03-14  
**测试范围**: 启动时间、内存占用、地图加载、离线存储

---

## 一、测试目标

1. 建立APP性能基准指标
2. 识别性能瓶颈
3. 对比不同设备性能表现
4. 确保满足发布性能要求

## 三、测试基准定义

### 3.1 启动时间基准

| 指标 | 优秀 | 良好 | 可接受 | 不可接受 |
|------|------|------|--------|----------|
| 冷启动 | <2s | 2-3s | 3-5s | >5s |
| 热启动 | <0.5s | 0.5-1s | 1-2s | >2s |
| 首屏渲染 | <1s | 1-1.5s | 1.5-2s | >2s |

### 3.2 内存占用基准

| 指标 | 优秀 | 良好 | 可接受 | 不可接受 |
|------|------|------|--------|----------|
| 启动内存 | <100MB | 100-150MB | 150-200MB | >200MB |
| 导航内存 | <150MB | 150-200MB | 200-300MB | >300MB |
| 内存泄漏 | 无 | 轻微 | 可控 | 严重 |

### 3.3 地图加载基准

| 指标 | 优秀 | 良好 | 可接受 | 不可接受 |
|------|------|------|--------|----------|
| 地图首屏 | <1s | 1-2s | 2-3s | >3s |
| 路线渲染 | <500ms | 500ms-1s | 1-2s | >2s |
| 缩放流畅度 | 60FPS | 50-60FPS | 30-50FPS | <30FPS |

### 3.4 离线存储基准

| 指标 | 优秀 | 良好 | 可接受 | 不可接受 |
|------|------|------|--------|----------|
| 杭州市区离线包 | <200MB | 200-300MB | 300-500MB | >500MB |
| 下载速度 | <10分钟 | 10-20分钟 | 20-30分钟 | >30分钟 |
| 存储效率 | <1MB/km² | 1-2MB/km² | 2-3MB/km² | >3MB/km² |

## 四、测试用例

### 4.1 启动时间测试

#### PERF-001: 冷启动时间测试

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量APP完全关闭后启动时间 |
| **测试步骤** | 1. 完全关闭APP(从后台划掉)<br>2. 清除系统缓存(可选)<br>3. 使用计时器记录点击图标到首屏完全渲染时间<br>4. 重复5次取平均值 |
| **测量指标** | - 点击图标时间(T0)<br>- 启动页显示时间(T1)<br>- 首屏渲染完成时间(T2)<br>- 总启动时间 = T2 - T0 |
| **通过标准** | 总启动时间 < 3秒 |
| **优先级** | P0 |

#### PERF-002: 热启动时间测试

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量APP后台恢复时间 |
| **测试步骤** | 1. 打开APP，进入首页<br>2. 按Home键进入后台<br>3. 等待30秒<br>4. 重新打开APP<br>5. 记录恢复到首页时间 |
| **通过标准** | 热启动时间 < 1秒 |
| **优先级** | P0 |

#### PERF-003: 首次安装启动

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量首次安装后启动时间 |
| **测试步骤** | 1. 卸载APP并清除数据<br>2. 重新安装<br>3. 首次启动并计时<br>4. 记录到首页时间 |
| **通过标准** | 首次启动时间 < 5秒 |
| **优先级** | P1 |

---

### 4.2 内存占用测试

#### PERF-004: 启动内存占用

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量APP启动后内存占用 |
| **测试步骤** | 1. 使用Profiler监控内存<br>2. 启动APP<br>3. 等待首页完全加载<br>4. 记录内存占用<br>5. 触发GC，记录稳定后内存 |
| **测量指标** | - 初始内存<br>- 峰值内存<br>- 稳定内存 |
| **通过标准** | 稳定内存 < 150MB |
| **优先级** | P0 |

#### PERF-005: 导航内存占用

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量导航过程中内存占用 |
| **测试步骤** | 1. 启动APP，记录基准内存<br>2. 进入导航页面<br>3. 模拟导航5分钟<br>4. 记录内存占用变化<br>5. 退出导航，检查内存释放 |
| **测量指标** | - 导航峰值内存<br>- 内存增长趋势<br>- 退出后内存释放率 |
| **通过标准** | - 导航内存 < 200MB<br>- 无持续增长趋势<br>- 退出释放 > 80% |
| **优先级** | P0 |

#### PERF-006: 内存泄漏检测

| 属性 | 内容 |
|------|------|
| **测试目的** | 检测页面切换内存泄漏 |
| **测试步骤** | 1. 在首页记录内存基准<br>2. 重复以下操作10次:<br>   - 进入发现页<br>   - 进入详情页<br>   - 返回首页<br>3. 强制GC后记录内存<br>4. 对比基准内存 |
| **测量指标** | 内存增长量 |
| **通过标准** | 增长 < 10MB |
| **优先级** | P1 |

#### PERF-007: 长时间运行稳定性

| 属性 | 内容 |
|------|------|
| **测试目的** | 验证长时间运行内存稳定 |
| **测试步骤** | 1. 启动APP并保持前台<br>2. 每10分钟记录内存<br>3. 持续1小时<br>4. 分析内存趋势 |
| **通过标准** | 内存增长 < 20MB/小时 |
| **优先级** | P1 |

### 4.3 地图加载性能测试

#### PERF-008: 地图首屏加载

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量地图首次显示时间 |
| **测试步骤** | 1. 打开APP进入首页<br>2. 记录点击地图Tab时间<br>3. 记录地图瓦片首次显示时间<br>4. 记录地图完全加载时间 |
| **测量指标** | - 白屏时间<br>- 首瓦片时间<br>- 完整加载时间 |
| **通过标准** | 完整加载 < 2秒 |
| **优先级** | P0 |

#### PERF-009: 路线渲染性能

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量路线轨迹渲染性能 |
| **测试步骤** | 1. 进入路线详情页<br>2. 选择长距离路线(>10km)<br>3. 记录从点击到轨迹完全渲染时间<br>4. 使用DevTools检查掉帧情况 |
| **测量指标** | - 渲染时间<br>- 帧率(FPS)<br>- 卡顿次数 |
| **通过标准** | - 渲染 < 1秒<br>- FPS > 50 |
| **优先级** | P1 |

#### PERF-010: 地图缩放流畅度

| 属性 | 内容 |
|------|------|
| **测试目的** | 测试地图缩放操作流畅度 |
| **测试步骤** | 1. 显示地图<br>2. 快速缩放(双指缩放10次)<br>3. 记录FPS变化<br>4. 观察是否卡顿 |
| **测量指标** | 平均FPS、最低FPS |
| **通过标准** | 平均FPS > 50 |
| **优先级** | P1 |

#### PERF-011: 离线地图加载

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量离线地图加载速度 |
| **测试步骤** | 1. 下载杭州离线地图<br>2. 关闭网络<br>3. 打开地图页面<br>4. 记录加载时间 |
| **测量指标** | 离线地图加载时间 |
| **通过标准** | < 1秒 |
| **优先级** | P1 |

### 4.4 离线存储性能测试

#### PERF-012: 离线地图包大小

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量离线地图存储占用 |
| **测试步骤** | 1. 查看离线地图下载页面<br>2. 记录杭州市区离线包大小<br>3. 计算存储效率(大小/面积) |
| **测量指标** | - 离线包大小<br>- 存储效率 |
| **通过标准** | 杭州市区包 < 300MB |
| **优先级** | P1 |

#### PERF-013: 离线地图下载速度

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量离线地图下载速度 |
| **测试步骤** | 1. 使用WiFi网络<br>2. 开始下载杭州市区离线包<br>3. 记录下载完成时间<br>4. 计算平均下载速度 |
| **测量指标** | - 下载时间<br>- 平均速度 |
| **通过标准** | 下载时间 < 20分钟(WiFi) |
| **优先级** | P2 |

#### PERF-014: 离线数据查询性能

| 属性 | 内容 |
|------|------|
| **测试目的** | 测量离线状态下数据查询速度 |
| **测试步骤** | 1. 关闭网络<br>2. 进入发现页<br>3. 搜索路线<br>4. 记录搜索响应时间 |
| **测量指标** | 搜索响应时间 |
| **通过标准** | < 500ms |
| **优先级** | P1 |
