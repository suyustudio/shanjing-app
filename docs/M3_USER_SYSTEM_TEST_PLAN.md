### 3.3 Token刷新机制测试

#### TC-TOKEN-001: Token自动刷新

| 属性 | 内容 |
|------|------|
| **测试目的** | 验证Token过期自动刷新 |
| **前置条件** | 用户已登录 |
| **测试步骤** | 1. 登录APP，记录Token过期时间<br>2. 等待Token接近过期(或修改本地时间)<br>3. 发起需要登录的API请求(如：获取收藏列表)<br>4. 观察Token是否自动刷新 |
| **预期结果** | 1. 检测到Token即将过期<br>2. 自动调用刷新接口<br>3. 获取新Token<br>4. 原请求成功执行 |
| **通过标准** | 用户无感知完成Token刷新，请求成功 |
| **优先级** | P0 |

#### TC-TOKEN-002: Refresh Token过期处理

| 属性 | 内容 |
|------|------|
| **测试目的** | 验证Refresh Token过期后重新登录 |
| **测试步骤** | 1. 登录APP<br>2. 等待Refresh Token过期(或修改本地时间)<br>3. 发起需要登录的操作<br>4. 观察行为 |
| **预期结果** | 1. Refresh Token过期无法刷新<br>2. 提示"登录已过期，请重新登录"<br>3. 跳转登录页<br>4. 登录后恢复正常 |
| **通过标准** | 正确引导重新登录，不崩溃 |
| **优先级** | P1 |

#### TC-TOKEN-003: Token刷新失败重试

| 属性 | 内容 |
|------|------|
| **测试目的** | 验证Token刷新失败重试机制 |
| **测试步骤** | 1. 登录APP<br>2. 模拟网络异常(如：飞行模式)<br>3. 等待Token过期<br>4. 发起需要登录的请求<br>5. 恢复网络<br>6. 再次发起请求 |
| **预期结果** | 1. 刷新失败时缓存请求<br>2. 恢复网络后自动重试<br>3. 成功后返回结果 |
| **通过标准** | 网络恢复后自动恢复，无需重新操作 |
| **优先级** | P1 |

#### TC-TOKEN-004: 多设备Token管理
