# 山径APP - 分享功能详细设计

## 核心洞察

**分享不是功能，是用户的核心动机**

- 用户徒步很大程度上是为了**发朋友圈/小红书**
- "记录"是为了"炫耀"，"炫耀"是为了**认同和社交货币**
- 分享体验好坏，直接决定用户是否愿意传播

---

## 分享场景

### 场景1：行中分享（实时炫耀）
| 时机 | 内容 | 平台 |
|------|------|------|
| 到达观景点 | "我在XX，风景太美了"+定位+照片 | 朋友圈、小红书 |
| 完成里程碑 | "已走5km，还剩3km"+进度卡片 | 朋友圈 |
| 遇到有趣的事 | 拍照+语音/文字 | 小红书、微博 |

### 场景2：行后分享（完整回顾）
| 时机 | 内容 | 平台 |
|------|------|------|
| 完成路线 | 完整轨迹视频/长图 | 朋友圈、小红书 |
| 生成回忆 | "一周一山"合集 | 小红书、抖音 |
| 达成成就 | 徽章/里程碑展示 | 朋友圈 |

---

## 分享形式

### 形式1：智能生成视频（核心差异化）

**自动剪辑**：
- 选取轨迹中的精彩片段（自动识别停留点、观景点）
- 匹配节奏音乐（根据时长自动选曲）
- 叠加数据动画（里程、海拔、时长动态展示）
- 转场效果（地图→照片→数据→地图）

**模板选择**：
| 模板 | 风格 | 适合 |
|------|------|------|
| 诗意文艺 | 慢节奏、古风音乐、水墨滤镜 | 小红书 |
| 活力运动 | 快节奏、电子音乐、数据突出 | 朋友圈 |
| 极简记录 | 纯数据、无音乐、高级感 | 微博 |

**用户干预**：
- 可替换照片
- 可调整顺序
- 可更换音乐
- 可编辑文字

### 形式2：长图海报

**布局**：
```
┌─────────────────┐
│   山径LOGO      │
├─────────────────┤
│   封面照片      │
├─────────────────┤
│  路线名称+数据   │
│  九溪十八涧     │
│  8.5km · 3h    │
├─────────────────┤
│  轨迹地图       │
├─────────────────┤
│  照片墙（3-6张）│
├─────────────────┤
│  二维码：下载APP│
└─────────────────┘
```

**样式**：
- 多种配色主题（根据路线风景自动推荐）
- 字体选择
- 布局选择（紧凑/宽松）

### 形式3：实时卡片

**行中快速分享**：
- 当前位置截图+简单数据
- 一键生成，3秒内
- 适合朋友圈即时分享

## 平台适配

### 微信朋友圈
| 适配 | 说明 |
|------|------|
| 视频 | 15秒/30秒/60秒选项 |
| 图片 | 9张限制，智能选图 |
| 文案 | 提供文案模板，用户可改 |
| 定位 | 显示"山径APP"自定义地点 |

### 小红书
| 适配 | 说明 |
|------|------|
| 封面 | 3:4竖版，智能选取最佳照片 |
| 标题 | 提供爆款标题模板 |
| 标签 | 自动推荐#徒步 #杭州 #九溪十八涧 |
| 正文 | 提供文案框架，用户填充 |

### 微博
| 适配 | 说明 |
|------|------|
| 长图 | 支持超长图（9图拼接） |
| 视频 | 支持长视频 |
| 话题 | 自动关联#山径# #一周一山# |

## 分享激励

| 行为 | 激励 |
|------|------|
| 首次分享 | 送徽章"初次分享" |
| 分享获赞10+ | 送积分 |
| 分享带来新用户 | 双方送会员天数 |
| 连续分享3次 | 解锁专属模板 |

---

## 技术实现

### 视频生成
- 客户端生成（利用手机GPU）
- 模板引擎：After Effects → Lottie → 视频
- 或云端生成（成本高，延迟大）

### 图片生成
- Canvas绘制
- 服务端生成（推荐，统一质量）

|------|------|
