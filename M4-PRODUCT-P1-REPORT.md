## 3. 任务二：用户反馈入口设计

### 3.1 方案概述

基于 M4-USER-FEEDBACK.md 的完整方案，制定 M4 内测阶段简化版实现。

### 3.2 简化版功能范围

| 模块 | 完整版 | 简化版（M4） |
|------|--------|-------------|
| 入口位置 | 设置页 + 导航页 + 场景触发 | 仅设置页 |
| 问题分类 | 9个一级 + 10个二级 | 4个一级 |
| 表单字段 | 7个字段 | 3个字段（类型、描述、截图） |
| 严重程度 | 4级 | 无 |
| 后台管理 | 完整后台 | 列表查看 + 飞书通知 |

### 3.3 核心功能

**入口设计:**
- 位置：设置页 → "帮助与反馈"
- 子入口：问题反馈、功能建议、常见问题

**反馈表单:**
```
问题类型 *: [功能异常/性能问题/数据问题/其他]
问题描述 *: [文本输入，10-500字]
截图（选填）: [最多3张，每张<5MB]
[✓] 自动收集日志
```

**自动附加信息:**
- 设备信息（平台、系统版本、设备型号、APP版本）
- 用户信息（用户ID）
- 上下文（当前页面、时间戳）
- 日志片段（最近50行）

### 3.4 实现计划

| 阶段 | 功能 | 工时 | 优先级 |
|------|------|------|--------|
| Phase 1 | 基础表单 + 提交接口 | 8h | P0 |
| Phase 2 | 图片上传 + 飞书通知 | 4h | P1 |
| **总计** | | **12h** | |

## 4. 任务三：埋点实施验证

### 4.1 验证范围

基于 data-tracking-spec-v1.2.md，验证 6 个 P0 埋点事件：

| 事件名 | 事件类型 | 验证重点 |
|--------|---------|---------|
| `share_trail` | 功能埋点 | 分享成功后触发，参数完整 |
| `sos_trigger` | 功能埋点 | 倒计时结束后触发，含位置信息 |
| `sos_cancel` | 功能埋点 | 取消操作后触发，含取消原因 |
| `navigation_start` | 功能埋点 | 导航开始后触发，含路线信息 |
| `trail_complete` | 功能埋点 | 导航完成后触发，含统计数据 |
| `share_cancel` | 功能埋点 | 取消分享后触发 |

### 4.2 验证维度

| 维度 | 验证内容 | 通过标准 |
|------|---------|---------|
| **触发时机** | 事件在正确时机触发 | 100% 正确 |
| **参数完整** | 必填参数全部上报 | 100% 完整 |
| **参数正确** | 参数值符合规范 | 100% 正确 |
| **失败场景** | 取消/失败事件正确上报 | 正常触发 |
| **数据链路** | 上报成功并入库 | 成功率 > 95% |

### 4.3 验证计划

| 阶段 | 时间 | 内容 |
|------|------|------|
| 预验证 | 实地测试前 1 天 | 开发环境验证埋点触发和参数 |
| 实地验证 | 2026-04-03 | 真实场景验证所有 P0 埋点 |
| 数据检查 | 实地测试后 1 天 | 后台数据核对和质量问题修复 |

📄 **详见**: [M4-DATA-TRACKING-VERIFICATION-REPORT.md](./M4-DATA-TRACKING-VERIFICATION-REPORT.md)
