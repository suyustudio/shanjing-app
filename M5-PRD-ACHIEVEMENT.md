## 一、功能概述

### 1.1 功能名称
成就系统 (Achievement System)

### 1.2 功能定位
通过游戏化激励体系，鼓励用户持续探索、记录徒步历程、分享户外体验，增强用户粘性和产品活跃度。

### 1.3 核心价值
- **记录成长:** 可视化呈现用户的徒步历程
- **激励探索:** 通过目标驱动用户尝试更多路线
- **社交分享:** 满足用户展示和分享的需求

### 1.4 参考竞品
- **Strava:** 勋章系统设计，挑战活动
- **Keep:** 等级徽章，运动成就
- **AllTrails:** 完成记录，里程碑

## 五、成就解锁流程

### 5.1 解锁触发时机

| 成就类型 | 触发时机 | 计算方式 |
|----------|----------|----------|
| 首次徒步 | 完成一次路线记录 | 首次记录完成事件 |
| 里程累计 | 每次记录完成 | SUM(所有记录距离) |
| 路线收集 | 完成路线记录 | COUNT(DISTINCT 路线ID) |
| 连续打卡 | 每日定时检查 | 连续有记录的日期 |
| 分享达人 | 分享行为/收到点赞 | 实时计数 |

### 5.2 解锁通知设计

#### 弹窗样式
```
┌─────────────────────────────┐
│                             │
│      [徽章大图动画]         │
│                             │
│       🎉 恭喜解锁！         │
│                             │
│       [徽章图标]            │
│      「千里行者」           │
│                             │
│    累计徒步距离达到 500km   │
│                             │
│    [分享成就]  [查看详情]   │
│                             │
└─────────────────────────────┘
```

#### 动效规范
- 徽章从中心放大出现（scale 0 → 1，500ms，spring）
- 背景光效扩散（opacity 0 → 1 → 0，800ms）
- 文字淡入（opacity 0 → 1，300ms，delay 200ms）
- 按钮上滑出现（translateY 20 → 0，300ms，delay 400ms）

## 八、API 接口

### 8.1 获取成就列表
```
GET /api/v1/achievements
Response: Achievement[]
```

### 8.2 获取用户成就
```
GET /api/v1/users/{id}/achievements
Response: UserAchievement[]
```

### 8.3 获取用户进度
```
GET /api/v1/users/{id}/progress
Response: UserProgress
```

### 8.4 检查成就解锁 (服务端)
```
POST /api/v1/achievements/check
Body: { user_id, activity_type, data }
Response: { unlocked: Achievement[], progress: UserProgress }
```
