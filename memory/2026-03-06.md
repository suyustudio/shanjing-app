### Firebase Test Lab 问题排查
**时间**: 2026-03-06 14:25-14:26

**受影响构建:** Build #81, Build #82

**Build #81 错误信息:**
```
ERROR: (gcloud.firebase.test.android.run) Could not copy [./apk/app-release.apk] to [gs://test-lab-771fq9d1vh7nw-yajqjiiu1ipcq/***/81/]
ResponseError 403: github-actions-testlab@***.iam.gserviceaccount.com does not have storage.objects.create access
Permission 'storage.objects.create' denied on resource
```

**Build #82 错误信息:**
```
ERROR: (gcloud.firebase.test.android.run) Could not copy [./apk/app-release.apk] to [gs://test-lab-771fq9d1vh7nw-yajqjiiu1ipcq/***/82/]
ResponseError 403: github-actions-testlab@***.iam.gserviceaccount.com does not have storage.objects.create access
Permission 'storage.objects.create' denied on resource
```

**问题原因:**
Service Account 缺少 **Cloud Storage** 写入权限。Firebase Test Lab 需要上传 APK 到 GCS，但 Service Account 没有 `storage.objects.create` 权限。

**状态:** ⚠️ 未修复（Build #82 仍然失败）

**解决方案:**
在 Google Cloud IAM 中给 Service Account 添加角色：
- **Storage Object Admin** (`roles/storage.objectAdmin`) 或
- **Storage Object Creator** (`roles/storage.objectCreator`)

**操作步骤:**
1. 打开 [Google Cloud Console - IAM](https://console.cloud.google.com/iam-admin/iam)
2. 找到 Service Account: `github-actions-testlab@***.iam.gserviceaccount.com`
3. 添加角色: **Storage Object Admin**
4. 重新触发 Build #84 验证
