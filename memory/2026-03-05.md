# 2026-03-05 工作日志

## 关键进展

### 1. 高德地图集成调试（第4天，仍未解决）

**尝试的构建版本：** Build #45-#58

**关键发现：**
1. **Flutter 端必须设置隐私合规** - `AMapPrivacyStatement` 是强制要求
2. **JNI 崩溃错误** - `java_class == null in call to GetStaticMethodID`
3. **版本匹配问题** - 高德 SDK 版本与 Flutter 插件版本必须严格匹配

### 2. 崩溃日志分析

**错误信息：**
```
JNI DETECTED ERROR: java_class == null in call to GetStaticMethodID
from libAMapSDK_MAP_v9_2_0.so / libAMapSDK_MAP_v8_0_1.so
```

**根本原因：**
- SO 库加载时找不到对应的 Java 类
- Flutter 插件的 Java 代码与高德 SDK 不匹配
- 版本冲突导致 JNI 调用失败

### 3. 已验证的方案

| 方案 | 结果 | 说明 |
|------|------|------|
| 不手动添加 SDK 依赖 | ❌ 失败 | 插件自动处理的版本不匹配 |
| 3dmap:8.0.1 + location:5.5.1 | ❌ 失败 | JNI 崩溃 |
| 3dmap:8.1.0 + location:5.6.1 | ❌ 失败 | JNI 崩溃 |
| 3dmap:9.2.0 + location:6.4.9 | ❌ 失败 | JNI 崩溃 |
| Flutter 端添加隐私合规 | ✅ 已添加 | 但原生 SDK 仍崩溃 |

### 4. 正确的配置（根据官方文档）

**Flutter 端（已修复）：**
```dart
AMapWidget(
  apiKey: AMapApiKey(...),
  privacyStatement: const AMapPrivacyStatement(
    hasContains: true,
    hasShow: true,
    hasAgree: true,
  ),
  ...
)
```

**Android 端（问题所在）：**
- `amap_flutter_map: ^3.0.0` 插件内部依赖 `3dmap:8.1.0`
- 但手动添加的依赖可能与插件内部版本冲突
