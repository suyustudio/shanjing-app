   - `kimi-search`: ✅ enabled - 提供搜索/获取功能

3. **模型配置**:
   - **主模型**: `deepseek/deepseek-reasoner`
   - **备用模型**: `deepseek/deepseek-chat`, `kimi-coding/k2p5`
   - **memorySearch**: 使用 Kimi 远程 embedding 服务

### 潜在影响
- **kimi-claw 插件**: 可能维持与 Kimi 服务的连接，消耗 token（即使未使用模型）
- **kimi-search 工具**: 每次搜索/获取都会调用 Kimi API，消耗 token
- **memorySearch 远程**: 每次 memory 搜索都会调用 Kimi embedding API，消耗 token

### 建议方案
1. **临时禁用 kimi-claw 插件**（如不需要 Kimi 桥接功能）
2. **暂停使用 kimi-search/kimi-fetch 工具**（如需搜索，使用 web_search/web_fetch 替代）
3. **切换 memorySearch 到本地 embedding**（如有本地 embedding 模型）

### 下一步
- 与用户确认是否需要临时禁用插件
- 评估插件禁用对功能的影响
