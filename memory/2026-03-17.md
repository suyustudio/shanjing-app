## 今日关键事件

### 1. 高德 SDK 崩溃问题复发
**用户反馈**: "这个问题不是在M1阶段就遇到过了吗，为啥现在还出现"

**问题根因**:
- M1 阶段 (2026-03-03) 已解决的 `abiFilters` 配置在后续重构中丢失
- `build.gradle` 缺少 `ndk { abiFilters "arm64-v8a", "armeabi-v7a", "x86_64" }`
- 导致高德地图 SO 库无法正确打包，运行时崩溃

**修复内容**:
```gradle
// android/app/build.gradle
defaultConfig {
    // ...
    ndk {
        abiFilters "arm64-v8a", "armeabi-v7a", "x86_64"
    }
}

dependencies {
    implementation 'com.amap.api:3dmap:9.7.0'
    implementation 'com.amap.api:location:6.4.0'
}
```

## 教训与反思

1. **M1 修复为什么会丢失？**
   - 项目重构时 `flutter create` 覆盖了手动配置
   - 没有建立配置检查清单
   - 需要把关键配置写入文档并定期检查

2. **推送失败的影响**
   - 本地修复无法生效
   - 浪费多次构建资源
   - 需要建立备用的代码提交方案

3. **工作流混乱的代价**
   - 10个配置文件互相冲突
   - 难以确定哪个工作流在真正运行
   - 应该早期就保持简洁
