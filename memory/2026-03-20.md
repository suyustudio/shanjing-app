# 2026-03-20 导航功能修复记录

## 修复内容总结

### P0: 返回/关闭崩溃
**问题原因：**
- dispose 方法可能被多次调用
- WidgetsBinding 移除顺序问题
- 返回键没有确认对话框

**修复方案：**
1. 添加 `_isDisposing` 标志防止重复 dispose
2. 优化 dispose 顺序：
   - 先移除 WidgetsBinding Observer
   - 再取消定位订阅
   - 然后停止定位和 TTS
   - 最后调用 super.dispose()
3. 添加 `WillPopScope` 包裹 Scaffold，导航模式下弹出确认对话框

### P1: TTS 语音功能不可用
**问题原因：**
- TTS 初始化没有检查语言可用性
- 缺少错误处理

**修复方案：**
1. 添加 `isLanguageAvailable('zh-CN')` 检查
2. 添加 `_isTtsAvailable` 标志
3. 所有 TTS 操作前检查可用性
4. 完善 try-catch 错误处理

### P1: 导航状态刷新
**问题原因：**
- 状态更新时缺少 mounted 检查
- 可能存在竞态条件

**修复方案：**
1. 所有 setState 前检查 `mounted && !_isDisposing`
2. 所有回调函数中添加 `_navigationCompleted` 检查
3. 生命周期方法中增加保护

### P2: 导航流程重新设计
**新流程：**
1. 点击开始导航 → 进入预览模式
2. 获取当前位置 → 显示在地图上
3. 规划到路线起点的路径 → 虚线显示
4. 显示路径预览 → 包含距离、时间信息
5. 用户确认 → 点击"开始导航"
6. 进入导航状态 → 实际导航开始

**实现：**
- 新增 `NavigationMode` 枚举（preview/navigating）
- 预览模式显示：路线、起点、终点、距离信息
- 添加预览路径（虚线）从当前位置到路线起点
- 用户确认后才进入实际导航
