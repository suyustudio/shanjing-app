### 2. 发现的核心问题

#### Build #37 成功但运行时白屏
**症状:**
- APK 构建成功（20.2MB）
- 安装正常
- 启动后白屏/黑屏，进程在后台存在但界面展不开
- 应用名显示为"杭州旅游指南"（旧名称）

**根本原因分析:**
1. **APK 中缺少高德地图 SO 库** - 只有 `libapp.so` 和 `libflutter.so`，没有 `libAMapSDK_MAP_v*.so`
2. **Flutter 引擎初始化失败** - 可能是主题配置或高德 SDK 初始化阻塞
3. **AndroidManifest.xml 配置** - 看起来正确，但可能有隐藏问题

#### 构建过程中的问题
1. **YAML 语法错误** - 多次因 Python 多行字符串导致构建失败
2. **Gradle 依赖语法** - `implementation`/`api`/`compile` 版本兼容问题
3. **ndk 块位置** - 必须放在 `defaultConfig` 内部
4. **XML 格式错误** - sed 修改后的 AndroidManifest.xml 格式不正确

## 经验教训

1. **YAML 多行字符串是陷阱** - 使用 heredoc 时要特别小心缩进和引号
2. **Gradle 版本兼容性** - `flutter create` 可能生成旧版 Gradle 配置
3. **SO 库必须显式依赖** - Flutter 插件不会自动打包原生 SO 库
4. **XML 格式要严格验证** - sed 修改容易破坏 XML 结构
5. **Agent 并行测试有效** - Product Agent 快速给出了验收评估
