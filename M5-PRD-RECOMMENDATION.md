## 一、功能概述

### 1.1 功能名称
路线推荐算法 V1 (Trail Recommendation Engine V1)

### 1.2 功能定位
基于用户特征和路线属性的简单规则引擎，为用户提供个性化路线推荐，提升路线发现效率和用户体验。

### 1.3 核心价值
- **个性化:** 根据用户偏好推荐合适路线
- **发现性:** 帮助用户发现可能感兴趣的新路线
- **效率:** 减少用户筛选路线的时间成本

### 1.4 技术方案
- **无机器学习:** 基于规则的加权评分模型
- **5 因子排序:** 难度匹配、距离、评分、热度、新鲜度
- **可配置:** 权重参数支持动态调整

## 三、5 因子排序模型

### 3.1 因子定义

| 因子 | 权重 | 说明 | 数据来源 |
|------|------|------|----------|
| 难度匹配 | 25% | 与用户历史选择难度的匹配度 | 用户行为数据 |
| 距离 | 20% | 与用户当前位置的地理距离 | GPS / IP 定位 |
| 评分 | 20% | 路线的综合评分 | 用户评价数据 |
| 热度 | 20% | 近期完成人数和收藏数 | 平台行为数据 |
| 新鲜度 | 15% | 路线上线时间 | 路线创建时间 |

### 3.2 权重配置

```json
{
  "weights": {
    "difficulty_match": 0.25,
    "distance": 0.20,
    "rating": 0.20,
    "popularity": 0.20,
    "freshness": 0.15
  }
}
```

**V1 版本权重固定，V2 版本支持用户自定义偏好**

## 四、因子计算逻辑

### 4.1 因子 1: 难度匹配 (Difficulty Match)

**定义:** 衡量路线难度与用户偏好的匹配程度

**计算逻辑:**
```
用户偏好难度 = 用户历史完成路线的平均难度
难度匹配分 = 1 - |路线难度 - 用户偏好难度| / 最大难度差

难度等级映射:
- 简单 = 1
- 适中 = 2
- 困难 = 3
- 专家 = 4

示例:
用户偏好难度 = 2 (适中)
路线难度 = 2 (适中)
匹配分 = 1 - |2-2|/3 = 1.0 (完全匹配)

路线难度 = 4 (专家)
匹配分 = 1 - |4-2|/3 = 0.33 (不太匹配)
```

**特殊情况:**
- 新用户无历史数据：默认匹配分为 0.8
- 用户偏好范围较宽（完成过多种难度）：允许 ±1 的误差范围

### 4.2 因子 2: 距离 (Distance)

**定义:** 路线起点与用户当前位置的距离

**计算逻辑:**
```
距离分 = max(0, 1 - 实际距离 / 最大参考距离)

最大参考距离 = 100km (超出视为远距离)

距离分级:
- < 10km: 优秀 (0.9-1.0)
- 10-30km: 良好 (0.7-0.9)
- 30-50km: 一般 (0.5-0.7)
- 50-100km: 较远 (0.2-0.5)
- > 100km: 很远 (0-0.2)
```

**特殊情况:**
- 无法获取位置：默认距离分 0.5
- 用户设置偏好距离范围：在范围内加分

### 4.3 因子 3: 评分 (Rating)

**定义:** 路线的综合用户评分

**计算逻辑:**
```
评分分 = 路线平均评分 / 5.0

评分来源:
- 用户评价平均分 (权重 70%)
- 官方推荐评分 (权重 30%)

最低有效评价数 = 10
评价数 < 10 时，评分分 = 0.7 (保底分)
```

### 4.4 因子 4: 热度 (Popularity)

**定义:** 路线近期的受欢迎程度

**计算逻辑:**
```
热度分 = min(1.0, 近30天完成人数 / 100 + 收藏数 / 50)

热度指标:
- 近30天完成人数 (权重 60%)
- 收藏数 (权重 40%)

热度分级:
- > 80: 热门 (0.8-1.0)
- 50-80: 较热 (0.5-0.8)
- 20-50: 一般 (0.3-0.5)
- < 20: 冷门 (0-0.3)
```

### 4.5 因子 5: 新鲜度 (Freshness)

**定义:** 路线的新颖程度

**计算逻辑:**
```
新鲜度分 = max(0, 1 - 上线天数 / 90)

新鲜度分级:
- < 7天: 全新 (0.9-1.0)
- 7-30天: 较新 (0.7-0.9)
- 30-60天: 一般 (0.4-0.7)
- 60-90天: 较旧 (0.1-0.4)
- > 90天: 旧 (0-0.1)
```

## 五、综合评分计算

### 5.1 计算公式

```
综合得分 = Σ(因子分 × 因子权重)

即:
Score = 难度匹配 × 0.25 + 
        距离分 × 0.20 + 
        评分分 × 0.20 + 
        热度分 × 0.20 + 
        新鲜度分 × 0.15
```

### 5.2 计算示例

**示例路线:** 九溪十八涧

| 因子 | 原始值 | 因子分 | 权重 | 加权分 |
|------|--------|--------|------|--------|
| 难度匹配 | 难度=简单 | 0.9 | 0.25 | 0.225 |
| 距离 | 5km | 0.95 | 0.20 | 0.190 |
| 评分 | 4.8分 | 0.96 | 0.20 | 0.192 |
| 热度 | 120人/月 | 0.9 | 0.20 | 0.180 |
| 新鲜度 | 上线30天 | 0.67 | 0.15 | 0.101 |
| **合计** | - | - | **1.0** | **0.888** |

**匹配度展示:** 四舍五入为 **89% 匹配**

### 5.3 排序规则

1. 计算所有符合条件路线的综合得分
2. 按得分降序排列
3. 同分路线按热度 > 评分 > 距离排序
4. 返回 Top N 结果

## 六、推荐策略

### 6.1 首页推荐策略

**目标:** 综合展示多样性的优质路线

**策略:**
- 前 3 条：综合得分最高的路线
- 第 4-5 条：适当加入新鲜度高的新路线
- 避免连续展示同难度路线
- 至少包含 1 条附近路线

### 6.2 路线列表排序策略

**目标:** 在原有筛选条件下优化排序

**策略:**
- 用户选择筛选条件后，在结果内应用推荐排序
- 保留用户明确指定的筛选（难度、距离等）
- 在未指定排序方式时默认使用推荐排序

### 6.3 相似路线推荐策略

**目标:** 推荐与当前路线相似的路线

**策略:**
- 同难度等级路线优先
- 同区域路线优先
- 相似时长/距离路线优先
- 排除用户已完成路线

**相似度计算:**
```
相似度 = 难度相同(0.4) + 区域相同(0.3) + 时长接近(0.2) + 距离接近(0.1)
```

### 6.4 附近推荐策略

**目标:** 优先展示用户周边的路线

**策略:**
- 距离因子权重提升至 40%
- 仅显示 50km 范围内的路线
- 按距离由近到远排序

## 七、冷启动处理

### 7.1 新用户推荐

**场景:** 无历史数据的用户

**策略:**
- 使用默认权重配置
- 优先推荐热门路线
- 优先推荐附近路线
- 展示多样化的难度选择

**默认推荐列表:**
1. 综合热度最高的 3 条路线
2. 距离最近的 2 条路线
3. 最新上线的 2 条路线

### 7.2 新路线推荐

**场景:** 刚上线的路线无热度数据

**策略:**
- 新路线保护期（30天内）热度分保底 0.5
- 官方推荐路线额外加分
- 优先展示给活跃用户

## 八、API 接口

### 8.1 获取推荐路线

```
GET /api/v1/trails/recommendations

Parameters:
- scene: string (home, list, similar, nearby)
- lat: float (用户纬度, optional)
- lng: float (用户经度, optional)
- limit: int (返回数量, default 10)
- exclude_ids: string[] (排除的路线ID, optional)

Response:
{
  "trails": [
    {
      "id": "trail_001",
      "name": "九溪十八涧",
      "match_score": 0.89,
      "match_factors": {
        "difficulty": 0.9,
        "distance": 0.95,
        "rating": 0.96,
        "popularity": 0.9,
        "freshness": 0.67
      }
    }
  ]
}
```

### 8.2 获取相似路线

```
GET /api/v1/trails/{id}/similar

Parameters:
- limit: int (default 5)

Response:
{
  "trails": [...],
  "similarity_scores": [...]
}
```

### 8.3 更新推荐权重 (管理员)

```
POST /api/v1/admin/recommendation/weights

Body:
{
  "difficulty_match": 0.25,
  "distance": 0.20,
  "rating": 0.20,
  "popularity": 0.20,
  "freshness": 0.15
}
```
