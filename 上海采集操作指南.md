# 上海采集操作指南（下周三使用）

## 📱 应用程序操作步骤

### 1. 安装与启动
- 从 GitHub Actions 下载最新的 APK 文件（Build #247 成功后）
- 安装到安卓手机（需允许安装未知来源应用）
- 打开「山径」APP，进入「我的」页面

### 2. 开始第一次采集
1. **找到入口**：在「我的」页面顶部，看到蓝色卡片「📡 紧急采集入口（上海采集用）」
2. **点击开始**：点击蓝色按钮「开始采集」
3. **填写信息**：
   - **路线名称**：必填，如“古美公园环湖线”
   - **城市**：默认“上海”（如需修改可编辑）
   - **描述**：可选，如“测试采集，约半小时路程”
4. **授权权限**：
   - 点击“开始采集”后，会依次申请：
     - ✅ **定位权限**（必需）- 允许“山径”访问设备位置
     - ✅ **相机权限**（可选）- 如需拍照标记 POI
     - ✅ **存储权限**（必需）- 保存轨迹数据到本地
   - **全部点“允许”**

### 3. 录制过程中
- **GPS 状态**：顶部状态栏显示“🟢 录制中”，并有呼吸灯动画
- **轨迹记录**：每秒记录 1 个 GPS 点（弱信号时 5 秒 1 个）
- **添加标记点**：
  - **拍照标记**：点击左下角相机图标，可拍照或从相册选择
  - **POI 标记**：长按地图上任意位置，可添加文字标记点
- **暂停/继续**：双击底部圆形按钮可暂停/继续录制
- **电量监控**：电量低于 20% 会自动提示进入省电模式

### 4. 结束录制
1. 点击底部圆形按钮“结束录制”
2. 确认对话框，点击“确认结束”
3. 数据自动保存到本地

### 5. 导出数据
1. 回到「我的」页面，点击「我的采集」
2. 找到刚录制的路线，点击“导出数据”
3. 选择格式：
   - **GPX**：通用 GPS 轨迹格式（推荐）
   - **JSON**：原始数据格式（备用）
4. 等待导出完成，记下文件路径

## 💻 ADB 提取脚本

### 脚本位置
```
/root/.openclaw/workspace/adb_extract.sh
```

### 使用方法
```bash
# 1. 连接手机（确保 USB 调试已开启）
# 2. 运行脚本
chmod +x adb_extract.sh
./adb_extract.sh

# 或指定输出目录
./adb_extract.sh /path/to/output
```

### 脚本功能
1. **检查设备连接**：确认手机已通过 ADB 连接
2. **列出所有文件**：搜索所有 .gpx 和 .json 文件
3. **提取文件**：按日期分类保存到本地
4. **生成清单**：创建 index.html 文件，便于浏览

### 手动 ADB 命令（备用）
```bash
# 1. 列出所有文件
adb shell ls -la /data/data/com.suyustudio.shanjing/app_flutter/recordings_export/

# 2. 提取单个文件
adb pull /data/data/com.suyustudio.shanjing/app_flutter/recordings_export/文件名.gpx .

# 3. 提取所有文件
adb shell "find /data/data/com.suyustudio.shanjing/app_flutter/recordings_export/ -name '*.gpx' -o -name '*.json'" | xargs -I {} adb pull {} .
```

## 🔧 故障排除

### 问题 1：找不到紧急入口卡片
- **原因**：APK 版本不包含最新修复
- **解决**：确保安装的是 Build #247 或之后的版本

### 问题 2：权限申请失败
- **原因**：手机系统权限设置
- **解决**：
  1. 手机设置 → 应用 → 山径 → 权限
  2. 手动开启：定位、相机、存储权限

### 问题 3：GPS 信号弱
- **现象**：顶部显示“GPS 信号弱，精度可能降低”
- **解决**：
  - 尽量在开阔地带采集
  - 等待几秒，看信号是否恢复
  - 弱信号时仍会记录（5 秒 1 点）

### 问题 4：导出失败
- **现象**：导出后显示“导出失败”
- **解决**：
  1. 检查手机存储空间
  2. 重新授权存储权限
  3. 尝试导出为另一种格式

### 问题 5：ADB 无法连接
- **解决**：
  1. 手机开启“开发者选项”
  2. 开启“USB 调试”
  3. 连接时在手机上点“允许调试”

## 📝 采集注意事项

### 数据质量
1. **起点选择**：在路线实际起点开始录制
2. **保持移动**：静止时 GPS 点可能漂移，保持缓慢移动
3. **标记关键点**：路口、观景点、卫生间等用 POI 标记
4. **拍照辅助**：用照片记录路况、标识、风景

### 设备准备
1. **电量充足**：确保手机电量 > 50%
2. **网络关闭**：采集期间可关闭移动数据节省电量
3. **屏幕常亮**：设置手机屏幕常亮，避免熄屏停止录制
4. **携带充电宝**：半小时采集也应备电

### 安全事项
1. **注意路况**：行走时看路，避免只看手机
2. **保管设备**：防止手机掉落或被盗
3. **天气适应**：雨天注意防水，晴天注意防晒
