# IOS_OFFLINE_MAP_TEST - iOS 离线地图测试方案

> **文档版本**: v1.0  > **制定日期**: 2026-03-19  > **优先级**: P2

---

## 测试目标

验证 iOS 平台离线地图功能的完整性和性能，确保与 Android 平台体验一致。

## 测试范围

| 功能模块 | 测试内容 | 优先级 |
|----------|----------|--------|
| 离线包下载 | 下载流程、断点续传、存储管理 | P0 |
| 离线地图渲染 | 地图显示、缩放、旋转 | P0 |
| 后台定位 | 后台持续定位权限、精度保持 | P0 |
| 轨迹叠加 | 路线轨迹在离线地图显示 | P0 |
| POI 显示 | 离线状态下 POI 标记显示 | P1 |
| 高德 SDK 适配 | iOS 特定 API 兼容性 | P1 |

## iOS 特定考虑

### 权限模型差异

| 权限 | Android | iOS | 测试重点 |
|------|---------|-----|----------|
| 位置权限 | 运行时申请 | 始终/使用时/拒绝 | 多种授权场景 |
| 后台定位 | 单一权限 | 需要额外声明 | Info.plist 配置 |
| 存储权限 | 需要申请 | 沙盒自动管理 | 离线包存储路径 |
| 通知权限 | 需要申请 | 需要申请 | SOS 推送通知 |

### 后台定位配置

```xml
<!-- Info.plist -->
<key>NSLocationWhenInUseUsageDescription</key>
<string>需要位置权限来提供导航服务</string>

<key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
<string>后台导航需要持续定位权限</string>

<key>NSLocationAlwaysUsageDescription</key>
<string>后台导航需要持续定位权限</string>

<key>UIBackgroundModes</key>
<array>
    <string>location</string>
</array>
```
