## 1. 测试计划概述

### 1.1 背景

M3 阶段功能基本完成，Build #112 成功构建。M4 阶段核心目标是：
- **功能稳定**: 确保 P0 功能 100% 可用
- **性能达标**: 建立性能基准并监控
- **实地验证**: 在真实场景下验证功能
- **iOS 准备**: 为 iOS 版本测试做好准备

### 1.2 测试范围

| 模块 | 测试内容 | 优先级 |
|------|----------|--------|
| **埋点事件** | 4 个核心事件的触发与上报 | P0 |
| **分享功能** | E2E 测试，多平台分享卡片生成 | P0 |
| **SOS 功能** | E2E 测试，紧急求救流程 | P0 |
| **安全提醒** | 定时提醒、低电量提醒 | P0 |
| **性能测试** | 启动时间、内存、APK 大小 | P1 |
| **实地测试** | 西湖环湖线路线测试 | P1 |
| **iOS 测试** | 离线地图、兼容性矩阵 | P2 |

### 1.3 测试资源

| 资源类型 | 配置 | 数量 |
|----------|------|------|
| 测试设备(Android) | Android 10+ | 3 台 |
| 测试设备(iOS) | iPhone 12+ | 2 台 |
| 测试账号 | 微信测试号 | 5 个 |
| 测试场地 | 西湖环湖线 | 1 条 |
| 测试人员 | QA 工程师 | 2 人 |

### 1.4 测试周期

```
Week 1 (03/19-03/25): P0 测试用例补充 + 自动化脚本
Week 2 (03/26-04/01): 性能基准建立 + 实地测试准备
```

### 2.5 P0 - 埋点失败场景测试 ⭐新增

详见: [`qa/m4/test_cases/TC-ANALYTICS-FAILURE.md`](./test_cases/TC-ANALYTICS-FAILURE.md)

**覆盖范围**:
| 失败场景 | 测试内容 | 预期行为 |
|---------|---------|---------|
| 网络失败 | 弱网/断网时的埋点行为 | 本地队列缓存，网络恢复后批量上报 |
| 用户取消分享 | 分享面板中点击取消 | 不上报share_trail，上报share_cancel |
| 用户取消SOS | SOS倒计时期间点击取消 | 不上报sos_trigger，上报sos_cancel |
| API失败 | 后端返回4xx/5xx | 本地保留，按指数退避重试（最多5次） |
| 超时失败 | 请求超过10秒 | 标记为失败，计入重试队列 |
| 批量上报失败 | 本地缓存超过50条 | 自动触发批量上报，失败则保留 |

**关键验收点**:
- [ ] 网络断开时埋点不丢失
- [ ] 恢复网络后缓存数据自动上报
- [ ] 取消操作不上报成功事件
- [ ] 失败事件有独立的事件名
- [ ] 重试机制有效（最多5次）

### 2.6 P0 - SOS压力测试场景 ⭐新增

详见: [`qa/m4/test_cases/TC-SOS-STRESS.md`](./test_cases/TC-SOS-STRESS.md)

**覆盖范围**:
| 压力场景 | 测试内容 | 验收标准 |
|---------|---------|---------|
| 连续多次触发 | 5分钟内连续触发3次SOS | 每次都能正常发送，不丢数据 |
| 弱网环境 | 2G网络下触发SOS | 请求超时后降级为本地存储，提示用户 |
| 无信号场景 | 飞行模式下触发SOS | 本地保存求救信息，恢复信号后自动发送 |
| 电量极低 | 电量<5%时触发SOS | 优先保证SOS功能，其他功能降级 |
| 后台触发 | APP在后台时触发SOS | 能唤醒APP并执行SOS流程 |
| 多联系人压力 | 10个紧急联系人同时发送 | 不阻塞UI，异步发送，记录每个结果 |

**关键验收点**:
- [ ] 连续触发不崩溃
- [ ] 无网络时用户可感知降级处理
- [ ] 信号恢复后自动补发
- [ ] 低电量模式SOS可用
- [ ] 后台触发正常唤醒

### 2.7 P1 - 分享失败场景 ⭐新增

详见: [`qa/m4/test_cases/TC-SHARE-FAILURE.md`](./test_cases/TC-SHARE-FAILURE.md)

**覆盖范围**:
| 失败场景 | 测试内容 | 预期行为 |
|---------|---------|---------|
| 微信未安装 | 设备无微信APP时点击分享 | 提示"未安装微信"，引导下载或选择其他方式 |
| 分享取消 | 海报生成后点击取消 | 关闭分享面板，上报share_cancel事件 |
| 网络中断恢复 | 分享过程中断网 | 保存海报到本地，提示用户稍后手动分享 |
| 存储空间不足 | 保存海报时空间不足 | 提示清理空间，不上报save_local事件 |
| 海报生成失败 | 内存不足导致生成失败 | 提示重试，上报poster_generation_failed |
| 权限拒绝 | 存储权限被拒绝 | 引导用户开启权限，不上报失败事件 |

**关键验收点**:
- [ ] 微信未安装有友好提示
- [ ] 取消操作有对应埋点
- [ ] 网络中断可恢复分享
- [ ] 存储不足有明确提示
- [ ] 失败场景有独立埋点
