## 测试用例详情

### TC-SHARE-E2E-001: 小红书卡片生成 (3:4)

#### 基本信息

| 属性 | 值 |
|------|-----|
| 用例 ID | TC-SHARE-E2E-001 |
| 用例名称 | 小红书分享卡片生成验证 |
| 优先级 | P0 |
| 前置条件 | 1. 已登录 2. 有导航记录或正在导航 |

#### 测试步骤

**场景 1: 正常生成卡片**

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 完成一条路线导航 | 导航结束页显示 |
| 2 | 点击"生成分享图" | 进入分享页面 |
| 3 | 选择"小红书"模板 | 预览 3:4 卡片 |
| 4 | 检查卡片内容 | 包含路线名、照片、数据、标签 |
| 5 | 点击生成 | 生成高清图片(1080x1440) |

**场景 2: 卡片元素验证**

| 检查项 | 预期结果 |
|--------|----------|
| 路线名称 | 显示正确，字体 72-96px |
| 主照片 | 清晰，智能滤镜增强 |
| 数据条 | 里程/时长/爬升准确 |
| 诗意文案 | 根据路线类型匹配 |
| 标签云 | 5-6 个标签，#前缀 |
| 品牌区域 | Logo + 二维码可扫描 |
| 整体比例 | 1080x1440 (3:4) |

**场景 3: 多照片选择**

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 导航时拍摄 5 张照片 | 照片保存 |
| 2 | 进入分享页面 | 照片列表显示 |
| 3 | 选择 4 张照片 | 智能选图算法选择最佳 |
| 4 | 生成卡片 | 2x2 网格布局 |

#### 验收标准

- [ ] 卡片尺寸 1080x1440px
- [ ] 所有文字清晰可读
- [ ] 二维码可扫描识别
- [ ] 生成时间 < 3 秒
- [ ] 文件大小 < 2MB

## 自动化测试脚本

详见: [`../automation/share_e2e_test.dart`](../automation/share_e2e_test.dart)

### 核心测试流程

```dart
// 伪代码
test('小红书卡片生成', () async {
  // 1. 完成导航
  await completeNavigation(testRoute);
  
  // 2. 进入分享页面
  await tap(find.byKey('share_button'));
  await tap(find.text('小红书'));
  
  // 3. 验证预览
  expect(find.byKey('xhs_preview'), findsOneWidget);
  
  // 4. 生成卡片
  await tap(find.text('生成'));
  
  // 5. 验证输出
  final image = await getGeneratedImage();
  expect(image.width, 1080);
  expect(image.height, 1440);
});
```
