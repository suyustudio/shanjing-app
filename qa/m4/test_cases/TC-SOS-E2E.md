## 测试概览

### 测试目标
验证 SOS 紧急求救功能的完整流程，确保关键时刻 100% 可用。

### 测试范围

| 功能模块 | 测试内容 | 优先级 |
|----------|----------|--------|
| SOS 触发 | 按钮点击、二次确认 | P0 |
| 位置获取 | GPS 定位、坐标精度 | P0 |
| 消息发送 | 短信、推送发送机制 | P0 |
| 多联系人 | 同时发送给多位联系人 | P0 |
| 异常处理 | 无网络、发送失败重试 | P0 |
| 误触防护 | 二次确认、取消机制 | P0 |

### 测试环境

| 项目 | 配置 |
|------|------|
| App 版本 | Build #112+ |
| 网络环境 | WiFi / 4G / 无网络 / 弱信号 |
| 测试设备 | Android 12+ (GPS 功能正常) |
| 测试 SIM 卡 | 可发送短信的测试卡 |

#### 基本信息

| 用例 ID | TC-SOS-E2E-001 |
| 用例名称 | SOS 按钮触发流程验证 |
| 优先级 | P0 |
| 前置条件 | 1. 已登录 2. 已设置紧急联系人 3. 正在导航中 |

#### UI 检查

| 检查项 | 预期结果 |
|--------|----------|
| 按钮位置 | 导航页面右上角/底部显著位置 |
| 按钮样式 | 红色背景，白色 SOS 字样 |
| 按钮大小 | 最小 56x56dp，易于点击 |
| 视觉提示 | 可能有脉冲动画提示 |

#### 测试步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 开始任意路线导航 | 导航页面正常显示 |
| 2 | 检查 SOS 按钮 | 按钮可见，样式正确 |
| 3 | 点击 SOS 按钮 | 弹出二次确认框 |

#### 验收标准

- [ ] SOS 按钮在导航页面始终可见
- [ ] 点击后弹出二次确认框
- [ ] 取消后返回导航无影响
- [ ] 确认后开始发送流程

## 自动化测试脚本

详见: [`../automation/sos_e2e_test.dart`](../automation/sos_e2e_test.dart)

### 核心测试流程

```dart
// 伪代码
test('SOS 完整流程', () async {
  // 1. 设置紧急联系人
  await setEmergencyContact(testContact);
  
  // 2. 开始导航
  await startNavigation(testRoute);
  
  // 3. 触发 SOS
  await tap(find.byKey('sos_button'));
  expect(find.byKey('sos_confirm_dialog'), findsOneWidget);
  
  // 4. 确认发送
  await Future.delayed(Duration(seconds: 2)); // 防误触延迟
  await tap(find.text('确认'));
  
  // 5. 验证发送
  expect(find.text('求救信息已发送'), findsOneWidget);
  
  // 6. 验证短信内容
  final sms = await mockSmsServer.getLastSms();
  expect(sms.contains('紧急求救'), true);
  expect(sms.contains(testContact.name), true);
});
```
