## 测试概览

### 测试目标
验证安全提醒功能的及时性、准确性和用户可控性。

### 测试范围

| 提醒类型 | 触发条件 | 优先级 |
|----------|----------|--------|
| 定时安全提醒 | 每 30 分钟 | P0 |
| 低电量警告 | 电量 < 20% | P0 |
| 省电模式 | 电量 < 20% 自动激活 | P0 |
| 偏航提醒 | 偏离路线 > 30m | P0 |
| 信号弱提醒 | GPS 信号弱持续 60s | P1 |

### 测试环境

| 项目 | 配置 |
|------|------|
| App 版本 | Build #112+ |
| 设备电量 | 可控（模拟器或真机调试） |
| GPS 信号 | 可控环境 |

## 测试用例详情

### TC-SAFETY-001: 定时安全提醒 (30分钟)

#### 基本信息

| 属性 | 值 |
|------|-----|
| 用例 ID | TC-SAFETY-001 |
| 用例名称 | 定时安全提醒验证 |
| 优先级 | P0 |
| 前置条件 | 正在导航，安全提醒开关开启 |

#### 提醒规格

| 属性 | 规格 |
|------|------|
| 触发间隔 | 30 分钟 |
| 提醒方式 | 震动 + 声音 + 弹窗 |
| 震动模式 | 长震动 1 秒 |
| 声音 | 系统提示音 |
| 弹窗内容 | "您已徒步 30 分钟，一切正常吗？" |
| 操作选项 | "我很好" / "需要帮助" |

#### 测试步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 开始导航 | 导航正常 |
| 2 | 等待 30 分钟 | 或修改系统时间 |
| 3 | 检查提醒 | 震动 + 声音 + 弹窗 |
| 4 | 检查弹窗内容 | 内容正确 |
| 5 | 点击"我很好" | 弹窗关闭，记录响应 |
| 6 | 等待下一个 30 分钟 | 再次触发提醒 |

#### 验收标准

- [ ] 每 30 分钟准确触发
- [ ] 震动、声音、弹窗同时触发
- [ ] 点击"我很好"正常关闭
- [ ] 点击"需要帮助"触发 SOS 流程

## 自动化测试脚本

详见: [`../automation/safety_reminder_test.dart`](../automation/safety_reminder_test.dart)

### 核心测试流程

```dart
// 伪代码
test('定时安全提醒', () async {
  // 1. 开启安全提醒
  await enableSafetyReminder(true);
  
  // 2. 开始导航
  await startNavigation(testRoute);
  
  // 3. 模拟时间经过 30 分钟
  await simulateTimePassing(minutes: 30);
  
  // 4. 验证提醒触发
  expect(find.byKey('safety_reminder_dialog'), findsOneWidget);
  expect(find.text('您已徒步 30 分钟，一切正常吗？'), findsOneWidget);
  
  // 5. 验证震动和声音
  expect(await getVibratorStatus(), true);
});

test('低电量警告', () async {
  // 1. 开始导航
  await startNavigation(testRoute);
  
  // 2. 模拟电量降至 20%
  await setBatteryLevel(20);
  
  // 3. 验证警告触发
  expect(find.byKey('low_battery_warning'), findsOneWidget);
  
  // 4. 验证省电模式按钮
  await tap(find.text('启用省电模式'));
  expect(await isPowerSaveMode(), true);
});
```
