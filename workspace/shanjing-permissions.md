# 闪境 App 权限申请流程

## 定位权限

### 申请时机

| 场景 | 触发时机 | 说明 |
|------|----------|------|
| 首次启动 | 用户首次打开 App 时 | 引导用户开启定位，用于推荐附近的活动/地点 |
| 功能触发 | 用户点击「附近」或「地图」相关功能时 | 用户主动表达使用意图后申请 |
| 权限被拒绝后 | 用户再次进入需要定位的功能时 | 提供引导文案，跳转系统设置 |

### 申请文案

#### 1. 系统弹窗文案（iOS/Android 系统级）

```
标题：允许"闪境"访问您的位置吗？
副标题：用于发现附近的活动和推荐地点
```

#### 2. 自定义前置引导弹窗（推荐在系统弹窗之前显示）

**首次启动场景：**
```
标题：开启位置权限
正文：开启定位后，我们可以为您推荐附近的热门活动和地点，让探索更便捷。
按钮：立即开启 / 暂不开启
```

**功能触发场景：**
```
标题：需要位置权限
正文：查看附近活动需要获取您的位置信息，我们不会记录您的行踪轨迹。
按钮：允许 / 取消
```

#### 3. 权限被拒绝后的引导文案

```
标题：位置权限未开启
正文：您已关闭位置权限，无法查看附近的活动。如需开启，请前往系统设置。
按钮：去设置 / 知道了
```

### 文案原则

1. **明确用途** — 说明获取定位的具体用途（推荐附近活动）
2. **打消顾虑** — 强调不会记录行踪轨迹
3. **给予选择** — 提供"暂不开启"选项，不强制用户
4. **引导清晰** — 被拒绝后提供明确的设置入口
