## 1. 设计原则

### 1.1 核心原则

| 原则 | 描述 | 实现方式 |
|------|------|----------|
| **减少焦虑** | 避免过度强调危险，保持积极氛围 | 主色调使用绿色系，红色仅用于最终确认状态 |
| **防误触设计** | SOS触发需要明确的确认机制 | 长按+倒计时+二次确认三层防护 |
| **清晰层次** | 区分主动求救和被动安全监控 | 信息架构：监控状态 > 主动功能 > 设置管理 |
| **紧急可用** | 紧急情况下易于操作 | 大按钮、高对比度、震动反馈 |

### 1.2 竞品洞察

**AllTrails**
- 无显性SOS按钮，重点在位置分享
- 设计启示：将Lifeline作为核心功能，SOS作为最后的保障

**Garmin**
- 物理按钮 + 保护盖，工业设计感
- 设计启示：软件上模拟"保护盖"概念，增加触发门槛

**Apple**
- 简洁的倒计时界面，清晰的取消机制
- 设计启示：倒计时界面要极简，取消操作要显眼

## 3. SOS紧急求助

### 3.1 设计策略

SOS功能采用**"渐进式揭示"**设计：
- 平时：折叠状态，减少视觉压力
- 展开后：明确的使用指引
- 触发时：极简界面，防止误操作

### 3.2 交互流程

```
┌─────────┐    点击展开    ┌─────────┐    长按按钮    ┌─────────┐
│  折叠态  │ ────────────▶ │  展开态  │ ────────────▶ │ 倒计时  │
│         │               │         │   (3000ms)    │         │
└─────────┘               └─────────┘               └────┬────┘
                                                         │
                    ┌────────────────────────────────────┘
                    │ 倒计时完成
                    ▼
┌─────────┐    点击确认    ┌─────────┐    发送完成    ┌─────────┐
│ 发送成功 │ ◀──────────── │ 确认弹窗 │ ────────────▶ │ 发送中  │
│         │               │         │               │         │
└─────────┘               └─────────┘               └─────────┘
     │
     │ 10秒后自动关闭 / 点击关闭
     ▼
┌─────────┐
│ 返回页面 │
└─────────┘
```

### 3.3 SOS折叠态

```
┌─────────────────────────────────┐
│  🆘 紧急求助              [▶]   │  ← 绿色边框，非红色
│  仅在真正紧急情况下使用          │
└─────────────────────────────────┘
```

**设计要点：**
- 使用绿色边框（`#2D968A`），表示"安全保障"

设计要点：

## 6. 视觉规范

### 6.1 颜色系统

#### 主色调

| 名称 | 亮色模式 | 暗黑模式 | 用途 |
|------|----------|----------|------|
| Primary | `#2D968A` | `#4DB6AC` | 主按钮、强调 |
| Primary Light | `#4DB6AC` | `#80CBC4` | 悬停、次要强调 |
| Primary Dark | `#1F6B62` | `#2D968A` | 按下状态 |

#### 功能色

| 名称 | 亮色模式 | 暗黑模式 | 用途 |
|------|----------|----------|------|
| Success | `#4CAF50` | `#66BB6A` | 成功状态、安全提示 |
| Warning | `#FF9800` | `#FFB74D` | 警告、超时提醒 |
| Error | `#F44336` | `#E57373` | 错误、SOS、紧急情况 |
| Emergency | `#D32F2F` | `#EF5350` | 纯紧急按钮 |

#### SOS专用渐变色

```css
/* SOS按钮渐变 - 仅在紧急时刻使用 */
SOS-Gradient: linear-gradient(135deg, #FF5722 0%, #D32F2F 100%);

/* SOS倒计时脉冲 */
SOS-Pulse: radial-gradient(circle, rgba(244,67,54,0.3) 0%, transparent 70%);
```

### 6.2 字体规范

| 层级 | 大小 | 字重 | 用途 |
|------|------|------|------|
| 页面标题 | 20pt | Medium | 导航栏标题 |
| 卡片标题 | 17pt | Semibold | SOS、Lifeline标题 |
| 正文 | 15pt | Regular | 描述文字 |

| 倒计时数字 | 48pt | Bold | SOS倒计时 |

## 9. 无障碍设计

### 9.1 视觉无障碍

| 项目 | 规范 |
|------|------|
| 对比度 | 文字与背景对比度 ≥ 4.5:1 |
| SOS按钮 | 紧急状态对比度 ≥ 7:1 |
| 色盲友好 | 不仅依赖颜色传达状态，配合图标 |
| 动态字体 | 支持系统字体大小调整 |

### 9.2 操作无障碍

| 场景 | 设计 |
|------|------|
| 手套操作 | SOS按钮最小 88px，间距充足 |
| 单手操作 | 主要按钮位于屏幕下半区 |
| 视力辅助 | 所有图标配合文字标签 |
| 语音播报 | SOS触发时语音播报状态 |

### 9.3 无障碍标签

```dart
// SOS按钮
Semantics(
  label: 'SOS紧急求助按钮',
  hint: '长按3秒触发紧急求助，松开取消',
  button: true,
  onTapHint: '开始长按倒计时',
  child: SOSButton(),
)

// 倒计时界面
Semantics(
  label: 'SOS倒计时',
  value: '剩余$_countdown秒，松开手指取消',
  liveRegion: true,
  child: CountdownDisplay(),
)

// Lifeline状态
Semantics(
  label: '位置分享状态',
  value: _isSharing ? '分享中，预计返回$_returnTime' : '未开启',
  child: LifelineCard(),
)
```

### 9.4 触觉反馈

| 场景 | 反馈类型 |
|------|----------|
| 开始倒计时 | 短震动 (50ms) |
| 每秒倒计时 | 轻微震动 (20ms) |
| 确认弹窗 | 中等震动 (100ms) |
| 发送成功 | 成功震动 (200ms + 100ms) |
| 发送失败 | 错误震动 (100ms x 3) |

### 9.5 户外场景优化

| 场景 | 设计对策 |
|------|----------|
| 阳光直射 | 高对比度模式、加粗文字 |
| 低电量 | 简化动画、降低更新频率 |
| 手套操作 | 大点击区域 (min 44x44pt) |
| 湿手/雨滴 | 按钮间距加大、防误触 |
