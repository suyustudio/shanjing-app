## 1. 交互设计原则

### 1.1 设计目标

1. **流畅性** - 减少等待，即时反馈
2. **一致性** - 相同操作，相同反馈
3. **可预测** - 用户能预判操作结果
4. **容错性** - 允许撤销，防止误操作

### 1.2 响应时间规范

| 操作类型 | 最大响应时间 | 反馈方式 |
|----------|--------------|----------|
| 即时反馈 | 100ms | 视觉变化 |
| 快速响应 | 300ms | 转场动画 |
| 正常响应 | 1s | 加载指示 |
| 慢速操作 | 1s+ | 进度提示 |

## 2. 评论区交互

### 2.1 评论列表

#### 列表加载

| 场景 | 交互 | 反馈 |
|------|------|------|
| 首次进入 | 骨架屏 → 内容 | 2s内显示内容 |
| 下拉刷新 | 下拉 → 释放 → 刷新 | 顶部loading + 数据更新 |
| 上拉加载 | 底部触发 → 加载更多 | 底部loading + 追加数据 |
| 无更多数据 | 底部提示 | "没有更多评论了" |

#### 评论项交互

| 操作 | 触发 | 反馈 | 结果 |
|------|------|------|------|
| 展开评论 | 点击"展开" | 文字展开，按钮变"收起" | 显示完整内容 |
| 收起评论 | 点击"收起" | 文字收起，按钮变"展开" | 收起多余内容 |
| 查看回复 | 点击"查看全部X条回复" | 展开回复列表 | 显示完整回复 |
| 点赞评论 | 点击👍 | 数字+1，图标变实心 | 发送点赞请求 |
| 取消点赞 | 再次点击👍 | 数字-1，图标变空心 | 取消点赞 |
| 回复评论 | 点击"回复" | 输入框聚焦，显示@对象 | 准备回复 |
| 长按评论 | 长按500ms | 弹出操作菜单 | 复制/举报/删除 |

#### 长按菜单

```
┌─────────────────┐
│ 📋 复制文字     │
├─────────────────┤
│ ↩️ 回复         │
├─────────────────┤
│ 🚨 举报         │  (自己的不显示)
├─────────────────┤
│ 🗑️ 删除         │  (仅自己的显示)
└─────────────────┘
```

### 2.2 评论输入

#### 输入框状态

| 状态 | 触发 | 视觉变化 |
|------|------|----------|
| 默认 | 无操作 | 灰色背景，占位文字 |
| 聚焦 | 点击输入框 | 白色背景，光标闪烁，键盘弹出 |
| 输入中 | 有文字 | 发送按钮可用，字数统计 |
| 回复模式 | 点击回复 | 输入框顶部显示回复对象 |

#### 键盘交互

| 操作 | 反馈 |
|------|------|
| 键盘弹出 | 输入框上移，页面内容上推 |
| 键盘收起 | 输入框下移，恢复全屏 |
| 点击表情 | 键盘切换为表情面板 |
| 点击空白 | 收起键盘 |

#### 发送流程

```
点击发送
    ↓
显示发送中（输入框loading）
    ↓
发送成功
    ↓
清空输入框 → 新评论插入列表顶部
    ↓
显示成功toast（可选）
```

### 2.3 排序切换

| 操作 | 反馈 | 结果 |
|------|------|------|
| 点击排序 | 下拉菜单展开 | 显示排序选项 |
| 选择选项 | 菜单收起，列表刷新 | 按新排序重载 |
| 点击空白 | 菜单收起 | 取消操作 |

## 6. 照片上传交互

### 6.1 照片选择

#### 选择流程

```
点击添加区域
    ↓
弹出选择器（相册/相机）
    ↓
选择照片
    ↓
照片显示在预览区（从左滑入）
    ↓
可继续选择（最多9张）
```

#### 相册选择器

| 操作 | 反馈 |
|------|------|
| 点击照片 | 选中标记出现，照片加入预览 |
| 再次点击 | 取消选中 |
| 已达上限 | 选中失败，toast提示"最多选择9张" |
| 点击完成 | 关闭选择器，返回编辑页 |

#### 相机拍摄

| 操作 | 反馈 |
|------|------|
| 点击拍照 | 快门动画，保存照片 |
| 确认使用 | 照片加入预览区 |
| 重新拍摄 | 返回相机预览 |

### 6.2 照片编辑

#### 删除照片

| 操作 | 反馈 |
|------|------|
| 点击删除按钮 | 照片缩小消失，后续照片左移补位 |
| 删除主图 | 自动将第一张设为主图 |

#### 设置主图

| 操作 | 反馈 |
|------|------|
| 长按照片 | 弹出菜单，选择"设为主图" |
| 设置成功 | 主图标记移动到该照片 |

#### 照片排序

| 操作 | 反馈 |
|------|------|
| 长按照片拖动 | 照片浮起，位置跟随手指 |
| 释放 | 照片固定到新位置 |

### 6.3 表单填写

#### 评分交互

| 操作 | 反馈 |
|------|------|
| 点击第n颗星 | 前n颗星依次点亮（带延迟） |
| 滑动评分 | 星星跟随滑动实时点亮 |
| 再次点击 | 可重新评分 |

#### 标签选择

| 操作 | 反馈 |
|------|------|
| 点击标签 | 标签背景变色，加入已选 |
| 再次点击 | 取消选中，恢复默认 |
| 点击添加 | 弹出输入框，输入新标签 |
| 新标签确认 | 新标签加入列表并选中 |

#### 位置选择

| 操作 | 反馈 |
|------|------|
| 点击位置栏 | 进入地图选择页 |
| 自动定位 | 显示当前位置，可微调 |
| 地图选点 | 移动地图，中心点定位 |
| 确认位置 | 返回编辑页，显示地址 |

### 6.4 发布流程

#### 发布验证

| 检查项 | 未通过提示 |
|--------|------------|
| 至少1张照片 | "请至少选择一张照片" |
| 网络连接 | "网络异常，请检查网络" |
| 内容审核 | "内容包含敏感信息" |

#### 发布进度

```
点击发布
    ↓
验证表单
    ↓
显示上传进度（顶部toast）
    ↓
逐张上传照片
    ↓
提交表单数据
    ↓
发布成功
    ↓
返回上一页，显示成功toast
    ↓
新照片出现在列表顶部
```

#### 上传进度显示

| 进度 | 显示 |
|------|------|
| 0% | "准备上传..." |
| 1-99% | "正在上传 X/Y..." + 进度条 |
| 100% | "处理中..." |
| 完成 | 自动消失 |

#### 后台上传

| 场景 | 处理 |
|------|------|
| 用户返回 | 继续上传，完成后通知 |
| APP关闭 | 上传暂停，重新打开继续 |
| 上传失败 | 保存草稿，提示重新上传 |

### 6.5 草稿保存

| 触发条件 | 操作 |
|----------|------|
| 用户点击取消 | 提示"是否保存草稿" |
| APP意外关闭 | 自动保存草稿 |
| 上传失败 | 自动保存草稿 |

#### 草稿恢复

```
重新进入上传页
    ↓
检测到有草稿
    ↓
弹出提示"是否恢复上次编辑？"
    ↓
[恢复编辑] / [丢弃]
```

| 交互 | 动画 | 时长 | 缓动 |

### 7.3 加载动画

| 场景 | 动画 | 时长 |
|------|------|------|

| 上传进度 | 进度条填充 | 根据实际上传 |

### 7.4 物理反馈

| 操作 | 反馈 |
|------|------|

| 快速滑动 | 惯性滚动 |

## 8. 异常处理

### 8.1 网络异常

| 场景 | 处理方式 |
|------|----------|
| 加载失败 | 显示重试按钮，保留已加载内容 |
| 发送失败 | 保留输入，提示重试 |
| 上传中断 | 保存进度，支持断点续传 |
| 弱网环境 | 压缩图片，降低质量优先上传 |

### 8.2 空状态

| 场景 | 显示内容 |
|------|----------|
| 无评论 | 空插画 + "快来发表第一条评论吧" |
| 无照片 | 空插画 + "还没有照片，去上传吧" |
| 无收藏 | 空插画 + "看到喜欢的内容就收藏吧" |
| 无搜索结果 | 空插画 + 搜索建议 |

### 8.3 权限处理

| 权限 | 拒绝处理 |
|------|----------|
| 相册访问 | 提示去设置开启，提供手动输入选项 |
| 相机访问 | 提示去设置开启 |
| 位置访问 | 允许手动选择位置 |
| 通知权限 | 非必须，下次提醒 |
