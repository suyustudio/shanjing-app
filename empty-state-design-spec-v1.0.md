# 山径APP - 空状态/异常状态设计规范 v1.0

> **文档版本**: v1.0  
> **制定日期**: 2026-02-28  
> **文档状态**: Week 3 Day 1 - P7  
> **适用范围**: 山径APP全功能空状态与异常状态  
> **产品定位**: 城市年轻人的轻度徒步向导

---

## 1. 设计原则

### 1.1 核心理念

空状态和异常状态不是"没有内容"，而是与用户沟通的重要触点。好的空状态设计应该：

| 原则 | 说明 | 设计表达 |
|------|------|----------|
| **清晰传达** | 让用户明白发生了什么 | 简洁明了的标题和说明 |
| **提供指引** | 告诉用户下一步可以做什么 | 明确的操作按钮或建议 |
| **保持品牌调性** | 即使在异常状态也传递品牌温度 | 符合山径自然、诗意的设计语言 |
| **减少焦虑** | 避免使用技术术语和负面词汇 | 温和的文案，安抚的插画 |

### 1.2 状态分类

| 大类 | 子类 | 触发场景 |
|------|------|----------|
| **空状态** | 无数据 | 列表为空、搜索结果为空 |
| | 无网络 | 设备未连接网络 |
| | 无权限 | 用户未授权必要权限 |
| **加载状态** | 初始加载 | 页面首次加载数据 |
| | 下拉刷新 | 用户主动刷新 |
| | 加载更多 | 列表分页加载 |
| | 操作等待 | 按钮点击后的处理中 |
| **错误状态** | 网络错误 | 请求超时、连接失败 |
| | 服务器错误 | 500错误、服务不可用 |
| | 客户端错误 | 404、参数错误 |
| | 业务错误 | 功能受限、版本过低 |

### 1.3 设计原则

1. **一致性** - 同类型状态保持视觉和交互一致
2. **及时反馈** - 状态变化立即呈现，不延迟
3. **可恢复性** - 提供明确的恢复路径
4. **容错性** - 优雅降级，保证核心功能可用

## 2. 空状态设计

### 2.1 无数据空状态

#### 使用场景
- 列表页没有任何数据
- 筛选/搜索结果为空
- 功能模块暂无内容

#### 设计规范

**视觉结构**
```
┌─────────────────────────────┐
│                             │
│        [插画/图标]           │  ← 64-120px，品牌色或中性色
│                             │
│      标题 (16-18px)          │  ← Medium字重
│                             │
│   说明文字 (14px, Gray-500)   │  ← 最多2行
│                             │
│      [操作按钮]              │  ← Primary按钮（可选）
│                             │
└─────────────────────────────┘
```

**具体场景设计**

| 场景 | 插画 | 标题 | 说明 | 操作 |
|------|------|------|------|------|
| **收藏为空** | 空盒子插画 | 还没有收藏 | 收藏喜欢的路线，随时查看 | 去探索路线 |
| **历史记录为空** | 脚印插画 | 还没有记录 | 开始你的第一次徒步之旅吧 | 开始记录 |
| **搜索结果为空** | 放大镜插画 | 没有找到相关路线 | 换个关键词试试，或浏览推荐路线 | 查看推荐 |
| **离线包为空** | 地图插画 | 还没有离线包 | 下载离线包，无网络也能导航 | 去下载 |
| **消息通知为空** | 信封插画 | 还没有消息 | 有新消息时会通知你 | - |

**文案规范**
- 标题：简短有力，不超过10个字
- 说明：解释原因或提供建议，不超过30个字
- 语气：温和鼓励，避免责备用户

**设计稿描述**
```
收藏页空状态:
- 背景: White (#FFFFFF)
- 插画: 空盒子插画，尺寸 100x100px，颜色 Primary-300 (#72C0B5)
- 标题: "还没有收藏"，16px, Medium, Gray-900
- 说明: "收藏喜欢的路线，随时查看"，14px, Regular, Gray-500
- 按钮: "去探索路线"，Primary-500背景，White文字，圆角8px
- 垂直居中，上下间距 40px
```

### 2.3 无权限空状态

#### 使用场景
- 定位权限未授权
- 存储权限未授权
- 相机权限未授权（AR导航）
- 通知权限未授权

#### 设计规范

**视觉结构**
```
┌─────────────────────────────┐
│                             │
│        [权限图标]            │  ← 80px，Primary-400
│                             │
│      需要位置权限            │  ← 18px, Medium
│                             │
│   开启位置权限后，才能使用     │  ← 14px, Gray-500
│   导航和路线推荐功能          │
│                             │
│      [开启权限]              │  ← Primary按钮
│                             │
└─────────────────────────────┘
```

**各权限场景设计**

| 权限类型 | 图标 | 标题 | 说明 | 操作 |
|----------|------|------|------|------|
| **定位权限** | 定位图标 | 需要位置权限 | 开启位置权限后，才能使用导航和路线推荐功能 | 开启权限 |
| **存储权限** | 文件夹图标 | 需要存储权限 | 用于保存离线地图和路线记录数据 | 开启权限 |
| **相机权限** | 相机图标 | 需要相机权限 | 用于AR实景导航功能 | 开启权限 |
| **通知权限** | 铃铛图标 | 开启消息通知 | 及时获取路线推荐和安全提醒 | 开启通知 |

**权限说明文案**
- 必须说明"为什么需要这个权限"
- 强调权限与功能的关联
- 避免使用"否则无法使用"等威胁性语言

**设计稿描述**
```
定位权限空状态:
- 背景: White (#FFFFFF)
- 图标: 定位图标，80x80px，Primary-400 (#4FABA0)
- 标题: "需要位置权限"，18px, Medium, Gray-900
- 说明: "开启位置权限后，才能使用导航和路线推荐功能"，14px, Regular, Gray-500
- 按钮: "开启权限"，Primary-500背景，White文字，圆角8px
- 底部提示: "你可以在系统设置中随时更改权限"，12px, Gray-400
```

## 3. 加载状态设计

### 3.1 骨架屏 (Skeleton)

#### 使用场景
- 页面首次加载
- 内容结构已知但数据未返回
- 列表/卡片内容加载

#### 设计规范

**基础样式**
```
骨架屏元素:
- 背景色: Gray-100 (#F3F4F6)
- 圆角: 4px（文字）/ 8px（卡片）
- 动画:  shimmer 流光动画
```

**Shimmer动画参数**
```
动画: 从左到右流光
- 起始: -100% 位置，透明
- 中间: 0% 位置，White 30% 透明度
- 结束: 100% 位置，透明
- 时长: 1.5s
- 循环: 无限循环
- 缓动: ease-in-out
```

### 3.2 加载动画

#### 使用场景
- 下拉刷新
- 加载更多
- 操作提交中
- 数据计算中

#### 设计规范

**下拉刷新动画**
```
样式:
- 高度: 60px
- 背景: White 或 Gray-50
- 指示器: 品牌色 Primary-500
- 文字: "下拉刷新" / "释放刷新" / "刷新中..."

动画:
- 下拉时: 箭头随下拉距离旋转
- 释放后: 旋转指示器，1s/圈
- 完成时: 对勾图标 + "刷新成功"
```

**加载更多动画**
```
样式:
- 高度: 50px
- 背景: 透明
- 指示器: 20px，Primary-500
- 文字: "正在加载更多..." / "没有更多了"

位置: 列表底部
```

**操作加载动画**
```
样式:
- 按钮内: 16px 旋转指示器，White
- 全屏: 48px 旋转指示器，Primary-500
- 遮罩: Black 30% 透明度

使用场景:
- 按钮点击后: 按钮内显示，禁用按钮
- 页面跳转前: 全屏显示，阻止操作
```

**加载指示器规格**
```
品牌加载指示器:
- 类型: 圆形进度条
- 颜色: Primary-500 (#2D968A)
- 线宽: 3px
- 尺寸: 小(20px) / 中(36px) / 大(48px)
- 动画: 360度旋转，1s/圈，linear

iOS风格:
- 使用系统UIActivityIndicatorView
- 颜色: Primary-500

Android风格:
- 使用CircularProgressIndicator
- 颜色: Primary-500
```

## 5. 特殊场景处理

### 5.1 弱网环境

**设计策略**
- 优先显示缓存内容
- 非关键数据延迟加载
- 提供明确的网络状态提示

**弱网提示条**
```
位置: 页面顶部
样式:
- 高度: 36px
- 背景: Warning-50 (#FFF8E1)
- 图标: 信号弱图标，16px，Warning-500
- 文字: "当前网络较慢，已显示缓存内容"，14px，Warning-700
- 操作: "刷新"按钮
```

### 5.2 离线模式

**设计策略**
- 明确标识离线状态
- 区分在线/离线功能
- 提供离线可用功能入口

**离线状态栏**
```
位置: 状态栏下方
样式:
- 高度: 32px
- 背景: Info-50 (#EEF7FF)
- 图标: 离线图标，14px，Info-500
- 文字: "离线模式"，13px，Info-700
```

### 5.3 部分加载失败

**设计策略**
- 成功内容正常显示
- 失败部分显示占位或重试按钮
- 不影响其他功能使用

**局部错误处理**
```
场景: 列表中某张图片加载失败
处理:
- 显示品牌色占位图
- 点击可重新加载
- 文字: "图片加载失败，点击重试"
```

## 6. 交互规范

### 6.1 状态切换动画

| 切换场景 | 动画效果 | 时长 |
|----------|----------|------|
| 加载 → 成功 | 骨架屏淡出，内容淡入 | 300ms |
| 加载 → 空状态 | 骨架屏淡出，空状态淡入 | 300ms |
| 加载 → 错误 | 骨架屏淡出，错误状态淡入 | 300ms |
| 错误 → 加载 | 错误状态淡出，骨架屏淡入 | 200ms |
| 下拉刷新 | 弹性回弹 + 内容更新 | 400ms |

### 6.2 重试机制

| 场景 | 重试策略 | 最大重试次数 |
|------|----------|--------------|
| 页面加载 | 自动重试 + 手动重试 | 2次自动 |
| 图片加载 | 自动重试 | 3次 |
| 提交操作 | 手动重试 | - |
| 导航数据 | 自动重试 | 无限（间隔5s） |

### 6.3 超时处理

| 操作类型 | 超时时间 | 超时提示 |
|----------|----------|----------|
| 页面加载 | 10s | "加载超时，请重试" |
| 下拉刷新 | 10s | "刷新超时" |
| 按钮操作 | 15s | "操作超时，请重试" |
| 文件下载 | 60s | "下载超时，请检查网络" |
