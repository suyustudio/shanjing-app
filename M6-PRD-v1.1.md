## 一、功能概述

### 1.1 功能模块
M6 阶段聚焦社交功能建设，包含四大核心模块：

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| 评论系统 | 路线评论、评分、回复互动、点赞 | P0 |
| 照片分享 | 徒步照片上传、瀑布流展示 | P0 |

| 收藏夹 | 路线分类收藏管理 | P0 |

### 1.2 功能定位
将山径从工具型应用升级为**社区化户外平台**，增强用户粘性和内容沉淀。

### 1.3 核心价值
- **内容沉淀:** 用户产生内容(UGC)丰富路线信息
- **决策辅助:** 真实评价帮助用户选择路线
- **社交激励:** 关注和互动提升用户活跃度
- **个性化管理:** 收藏夹帮助用户规划徒步计划

### 1.4 参考竞品
- **AllTrails:** 评论和照片系统
- **小红书:** 瀑布流和社区互动
- **Strava:** 关注系统和动态流

### 2.4 点赞功能详细设计

#### 2.4.1 点赞交互流程

```
用户点击点赞按钮
      ↓
前端调用 POST /v1/reviews/{reviewId}/like
      ↓
后端检查用户是否已点赞
      ↓
├─ 已点赞 → 返回 409 错误 (ALREADY_LIKED)
└─ 未点赞 → 创建 like 记录
              ↓
        更新 review.likeCount (+1)
              ↓
        返回更新后的 likeCount 和 isLiked=true
              ↓
        前端更新 UI（按钮高亮 + 数量+1）
```

#### 2.4.2 取消点赞流程

```
用户点击已点赞按钮
      ↓
前端调用 DELETE /v1/reviews/{reviewId}/like
      ↓
后端删除 like 记录
      ↓
更新 review.likeCount (-1)
      ↓
返回更新后的 likeCount 和 isLiked=false
      ↓
前端更新 UI（按钮恢复灰色 + 数量-1）
```

#### 2.4.3 点赞接口定义

```yaml
# 点赞评论
POST /v1/reviews/{reviewId}/like
Authorization: Bearer {token}

Response 200:
  success: true
  data:
    reviewId: string
    likeCount: number      # 更新后的点赞数
    isLiked: true

# 取消点赞
DELETE /v1/reviews/{reviewId}/like
Authorization: Bearer {token}

Response 200:
  success: true
  data:
    reviewId: string
    likeCount: number      # 更新后的点赞数
    isLiked: false

# 批量检查点赞状态（用于列表）
POST /v1/reviews/batch-check-like
Authorization: Bearer {token}
Request Body:
  reviewIds: string[]     # 最多 100 个

Response 200:
  success: true
  data:
    reviewId1: true
    reviewId2: false
    ...
```

### 2.6 评分计算规则

| 维度 | 规则 | 说明 |
|------|------|------|
| 总体评分 | 1-5 整数 | 用户直接给出的评分 |
| 平均分计算 | 算术平均 | `SUM(rating) / COUNT(reviews)` |
| 统计更新 | 实时更新 | 评论增删时同步更新路线统计 |

**注意**: 暂不使用复杂加权算法（去掉最高最低、30天权重），采用简单算术平均。

### 2.7 "体验过"认证标识

**认证逻辑**:
- 用户完成过该路线（有完成的徒步记录）
- 发表评论时自动标记 `isVerified=true`

**标识展示**:
- 评论列表：用户名旁显示 ✓体验过
- 标识颜色：主题色 #2D968A

### 2.8 评价标签体系

| 类别 | 标签 | 描述 |
|------|------|------|
| 风景 | 风景优美 | 自然景观出色 |
| 风景 | 拍照圣地 | 适合摄影 |
| 风景 | 秋色迷人 | 秋季推荐 |
| 风景 | 春花烂漫 | 春季推荐 |
| 难度 | 适合新手 | 入门级难度 |
| 难度 | 难度适中 | 需要一定体力 |
| 难度 | 挑战性强 | 较难路线 |
| 设施 | 设施完善 | 休息点/厕所充足 |
| 设施 | 补给方便 | 沿途可购买水/食物 |
| 人群 | 人少清静 | 避世好去处 |
| 人群 | 适合亲子 | 适合带孩子 |

### 2.9 评论审核机制

**敏感词过滤**:
- 发表时自动检测敏感词
- 包含敏感词的评论自动标记为 `pending` 状态
- 管理员审核后显示

**举报机制**:
- 用户可举报违规评论
- 举报原因：垃圾广告、人身攻击、虚假信息、其他
- 达到 3 次举报自动隐藏评论

### 3.2 照片上传流程（OSS直传）

```
1. 前端请求上传凭证
   POST /v1/photos/upload
   Request: { count: 3, fileType: "image/jpeg" }
   
2. 后端返回上传凭证
   Response: {
     uploadId: "upload_xxx",
     uploadUrl: "https://oss.aliyuncs.com",
     formData: { key, policy, signature, ... },
     previewUrl: "https://cdn.shanjing.app/photos/xxx.jpg"
   }

3. 前端直传 OSS
   POST https://oss.aliyuncs.com
   FormData: { ...formData, file }

4. 上传完成后创建记录
   POST /v1/photos
   Request: {
     uploadIds: ["upload_xxx"],
     trailId: "trail_xxx",      // 可选
     location: { lat, lng, name }, // 可选
     description: "...",        // 可选
     isPublic: true             // 默认 true
   }
```

### 3.3 照片上传规范

| 项目 | 规范 |
|------|------|
| 格式 | JPG, PNG, HEIC |
| 单张大小 | 最大 10MB |
| 单次数量 | 最多 9 张 |
| 分辨率 | 建议 ≥1080px |
| 压缩 | 上传后 OSS 自动压缩 |
| 存储 | 阿里云 OSS |
| CDN | 阿里云 CDN |
