## 1. 产品-设计对齐总览

### 1.1 对齐检查总结

基于产品策略文档（v1.0）对四个平台的分享策略定义，本次设计优化重点解决以下对齐问题：

| 平台 | 产品定义核心 | 设计优化重点 | 对齐状态 |
|------|-------------|-------------|---------|
| **小红书** | 精美情绪化、实用收藏、标签搜索 | 场景模板视觉化、文案排版优化、标签层级设计 | ✅ 已对齐 |
| **朋友圈** | 简洁数据、真实记录、隐私可控 | 数据可视化增强、模板差异化、隐私引导设计 | ✅ 已对齐 |
| **公众号** | 深度攻略、杂志排版、关注转化 | 信息层级优化、引导关注组件、长图适配 | ✅ 已对齐 |
| **微博** | 热点话题、互动裂变、九宫格 | 话题标签视觉化、互动引导设计、九宫格布局 | ✅ 已对齐 |

### 1.2 设计优化原则

**原则1：产品策略可落地**
- 每个产品定义的"场景模板"都有对应的视觉设计
- 文案风格体系转化为字体/排版规范
- 数据展示策略转化为可视化组件

**原则2：平台特性差异化**
- 严格遵循各平台的内容消费特征
- 适配各平台的图片规格和交互习惯
- 尊重各平台的用户心理预期

**原则3：品牌一致性**
- 保持山径APP品牌色彩系统
- 统一的数据可视化语言
- 一致的品牌露出规范

## 2. 小红书平台设计优化

### 2.1 场景模板 vs 视觉设计对齐

产品策略定义了5种场景模板，每种模板对应不同的视觉风格：

| 场景模板 | 产品定义 | 视觉设计规范 | 配色方案 |
|---------|---------|-------------|---------|
| **治愈系风景** | 青绿色调、清新自然、诗意文案 | 背景：青绿渐变叠加；文案：居中/左对齐；数据：半透明悬浮卡片 | 主色：#3D7A5D（青绿）；辅助：#5B8BA0（晨雾蓝） |
| **挑战成就** | 暖金色调、壮丽山景、励志文案 | 背景：暖金渐变；文案：顶部叠加；数据：突出里程+爬升 | 主色：#D4A574（暖沙）；辅助：#E8913A（落日橙） |
| **亲子时光** | 明亮暖调、温馨活泼、教育文案 | 背景：明亮暖黄；文案：底部居中；数据：圆角卡片 | 主色：#F5D76E（暖黄）；辅助：#E8913A（活力橙） |
| **闺蜜出行** | 清新粉调、ins风、社交文案 | 背景：清新粉紫渐变；文案：灵活布局；数据：极简展示 | 主色：#E8B4B8（柔粉）；辅助：#D4A5A5（粉紫） |
| **独行日记** | 暖紫色调、孤独美感、introspective文案 | 背景：暖紫渐变+暗角；文案：底部叠加；数据：低调处理 | 主色：#9B8AA5（暖紫）；辅助：#6B5B7A（深紫） |

### 2.2 文案风格 vs 字体排版对齐

产品策略定义了3种文案风格，对应不同的字体排版规范：

#### 诗意文艺型
```
字体层级：
- 路线名称：思源宋体，72-96px，Bold，白色带阴影
- 地点信息：苹方，32px，Regular，白色/浅灰
- 诗意文案：苹方，32px，Medium，白色，居中或左对齐
- 标签：苹方，28px，Regular，品牌绿 #2D5A3D

排版规范：
- 文案位置：图片下方居中，或叠加在图片下方1/3处
- 最大行数：3行
- 字间距：+0.05em（增加呼吸感）
- 行高：1.6
```

#### 种草安利型
```
字体层级：
- 路线名称：思源宋体，80px，Bold
- 推荐语：苹方，36px，Medium，白色
- 卖点清单：苹方，28px，Regular，使用✅标记
- 行动号召：苹方，32px，Bold，落日橙 #E8913A

排版规范：
- 文案结构：标题→推荐语→3个卖点→行动号召
- 使用Emoji作为视觉锚点
- 数据条放在文案上方，增强可信度
```

#### 实用攻略型
```
字体层级：
- 路线名称：思源宋体，72px，Bold
- 信息项标题：苹方，28px，Bold，品牌绿
- 信息内容：苹方，26px，Regular，深灰
- 注意事项：苹方，24px，Regular，警告黄 #FFC107

排版规范：
- 使用卡片式信息分组
- 左对齐，统一缩进
- 图标+文字组合增强可读性
```

### 2.3 数据展示 vs 可视化组件对齐

产品策略定义的核心数据条设计：

```
┌─────────────────────────────────┐
│                                 │
│   5.2km    🕐 2h    ⛰️ 150m    │
│   ━━━━━    ━━━━━    ━━━━━      │
│    距离     用时     爬升        │
│                                 │
└─────────────────────────────────┘

组件规范：
- 背景：半透明卡片 rgba(255,255,255,0.85) 或品牌绿 rgba(45,90,61,0.9)
- 圆角：16px
- 内边距：24px 32px
- 数字字体：DIN Alternate，48px，Bold
- 单位字体：苹方，24px，Regular
- 图标大小：32px
- 间距：数据项之间 48px

变体设计：
1. 横向等分版：适用于数据均衡展示
2. 突出里程版：里程数字放大至72px，其他数据缩小
3. 竖向排列版：适用于空间有限的场景
```

### 2.4 标签云设计优化

产品策略定义的四级标签体系视觉化：

```
标签展示规范：

一级（品牌标签）：
- 样式：实心胶囊，品牌绿背景，白色文字
- 位置：标签云最后一个
- 示例：[山径APP]

二级（地点标签）：
- 样式：描边胶囊，品牌绿边框，品牌绿文字
- 优先级：最高，放在标签云前两位
- 示例：#九溪十八涧 #杭州

三级（场景标签）：
- 样式：纯文字，带#前缀，品牌绿颜色
- 示例：#治愈系风景 #周末去哪儿

四级（热点标签）：
- 样式：纯文字，带#前缀，晨雾蓝颜色（区分显示）
- 示例：#每周一山

布局规范：
- 每行2-3个标签
- 最多2行（6个标签）
- 行间距：16px
- 标签间距：12px
```

### 2.5 分享流程 vs 页面设计对齐

产品流程中的关键页面设计规范：

#### 模板选择页
```
布局：
- 顶部："选择分享风格"标题
- 中部：智能推荐卡片（大图预览）
- 下部：更多风格网格（2×3）

视觉规范：
- 推荐卡片：圆角16px，阴影md，选中状态带品牌绿边框
- 风格图标：48px，使用对应场景的配色
- 风格名称：14px，Medium
```

#### 内容编辑页
```
布局：
- 顶部：照片预览区（可点击更换）
- 中部：文案编辑区 + 风格切换标签
- 下部：标签编辑区 + Emoji快捷栏

视觉规范：
- 照片预览：3:4比例，圆角12px
- 文案输入框：白色背景，圆角8px，最小高度120px
- 风格标签：胶囊按钮，选中状态填充
- Emoji栏：横向滚动，32px图标
```

#### 预览页
```
布局：
- 全屏预览：模拟小红书展示效果
- 底部操作：保存/分享按钮

视觉规范：
- 预览框：圆角12px，阴影lg
- 模拟状态栏：小红书风格
- 操作按钮：主按钮样式，全宽
```

## 3. 朋友圈平台设计优化

### 3.1 场景模板 vs 视觉设计对齐

产品策略定义的3种模板视觉化：

| 模板 | 产品定义 | 视觉设计 | 适用场景 |
|------|---------|---------|---------|
| **数据控** | 科技感、深色背景、详细数据 | 深色渐变背景（#1A1A3D → #0D2847）；里程数字超大（160px）；多数据网格布局 | 运动爱好者、数据记录 |
| **风景优先** | 沉浸式、照片为主、数据悬浮 | 照片占70%高度；半透明数据卡片悬浮；极简文案 | 摄影爱好者、风景分享 |
| **简约派** | 纯色背景、居中数据、高级感 | 纯色或微渐变背景；大数字居中；Logo低调处理 | 低调用户、极简风格 |

产品策略强调"数据为主"，设计优化如下：

规范：
- 线条颜色：品牌绿或白色

## 5. 微博平台设计优化

### 5.1 场景模板 vs 视觉设计对齐

产品策略定义的5种模板视觉化：

| 模板 | 产品定义 | 视觉设计 | 配色方案 |
|------|---------|---------|---------|
| **热点话题型** | 热点感、话题标签醒目 | 顶部话题区（橙色）；微博蓝+亮橙配色；热点Emoji | 话题橙 #FF6B35；微博红 #E6162D |

## 8. 开发实现指南

### 8.1 技术实现建议

#### 图片生成方案

```javascript
// 推荐方案：HTML+CSS+html2canvas
import html2canvas from 'html2canvas';

async function generateShareCard(platform, scene, data) {
  // 1. 选择模板配置
  const config = getPlatformConfig(platform, scene);
  
  // 2. 应用数据到模板
  const template = renderTemplate(config, data);
  
  // 3. 生成图片
  const canvas = await html2canvas(template, {
    width: config.width,
    height: config.height,
    scale: 2, // 高清输出
    useCORS: true,
    backgroundColor: null,
    logging: false
  });
  
  // 4. 压缩优化
  const optimized = await optimizeImage(canvas, {
    quality: 0.9,
    maxSize: 2 * 1024 * 1024 // 2MB
  });
  
  return optimized;
}
```

| 优化项 | 目标值 | 实现方式 |
