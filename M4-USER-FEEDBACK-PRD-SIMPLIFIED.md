## 1. 概述

### 1.1 设计目标

建立轻量、高效的用户反馈收集体系，支持 M4 内测阶段的快速迭代。

| 目标维度 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| **收集效率** | 降低用户反馈门槛 | 反馈提交步骤 = 3 步 |
| **信息完整** | 收集足够多的诊断信息 | 自动附加设备/版本信息 |
| **处理效率** | 快速响应用户反馈 | 24h 内首次响应 |

### 1.2 方案范围（简化版）

| 范围 | 包含内容 | 不包含 |
|------|---------|--------|
| **应用内入口** | 设置页反馈入口 | 导航页悬浮按钮、场景化触发 |
| **反馈分类** | 问题类型（单选） | 二级分类、严重程度 |
| **内容收集** | 文字描述、截图 | 日志手动上传、语音反馈 |
| **后台管理** | 简单列表查看、飞书通知 | 数据统计、用户回访、知识库 |

### 3.2 表单字段规范

| 字段 | 类型 | 必填 | 验证规则 | 说明 |
|------|------|------|---------|------|
| `type` | 单选 | 是 | 选择分类 | 4个选项 |
| `description` | 文本 | 是 | 10-500字 | 富文本输入 |
| `images` | 图片 | 否 | 最多3张，每张<5MB | 支持拍照/相册 |
| `attach_log` | 开关 | 否 | 布尔值 | 默认开启 |

### 3.3 问题分类选项

| 分类ID | 分类名称 | 说明 | 示例 |
|--------|---------|------|------|
| `function` | 功能异常 | 功能无法正常使用 | 点击无反应、闪退 |
| `performance` | 性能问题 | 卡顿、慢、耗电 | 地图加载慢、发热 |
| `data` | 数据问题 | 路线/POI数据错误 | 坐标偏差、信息错误 |
| `other` | 其他 | 不属于以上类别 | 建议、咨询 |

## 4. 交互流程

### 4.1 用户操作流程

```
用户发现问题
    ↓
进入设置页 → 点击"帮助与反馈"
    ↓
选择"问题反馈"
    ↓
选择问题类型（4选1）
    ↓
填写问题描述（10-500字）
    ↓
[可选] 添加截图（最多3张）
    ↓
点击"提交"按钮
    ↓
显示提交成功提示
    ↓
返回设置页
```

### 4.2 提交按钮状态

| 状态 | 条件 | 表现 |
|------|------|------|
| **禁用** | 必填项未填写 | 灰色，不可点击 |
| **正常** | 必填项已填写 | 蓝色，可点击 |
| **加载中** | 提交中 | 显示loading，不可点击 |
| **成功** | 提交成功 | 显示成功toast，自动返回 |
| **失败** | 提交失败 | 显示错误提示，保留表单内容 |

### 4.3 错误处理

| 场景 | 处理方式 |
|------|---------|
| 网络失败 | 提示"网络异常，请检查后重试"，保留表单 |
| 图片过大 | 提示"单张图片不能超过5MB" |
| 描述过短 | 提示"请至少输入10个字" |
| 描述过长 | 提示"最多输入500个字" |
| 提交失败 | 提示"提交失败，请重试"，提供"重试"按钮 |

## 5. 后台处理

### 5.1 数据处理流程

```
用户提交反馈
    ↓
[客户端] 组装数据（表单 + 自动信息）
    ↓
[客户端] POST /api/feedback/submit
    ↓
[后端] 接收并验证数据
    ↓
[后端] 保存到数据库
    ↓
[后端] 发送飞书通知
    ↓
[后端] 返回成功响应
```

### 5.2 飞书通知格式

```
📮 新用户反馈

问题类型: 功能异常
用户描述: 导航时地图突然变成空白了...

设备: Android 13 (Xiaomi 13)
版本: 1.0.0+build140
用户: u_abc123
时间: 2026-03-19 14:30:25

[查看详情] [回复用户]
```

### 5.3 管理后台（极简版）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **反馈列表** | 按时间倒序展示所有反馈 | P0 |
| **反馈详情** | 查看完整信息，包括设备日志 | P0 |
| **状态标记** | 标记为"待处理"/"处理中"/"已解决" | P1 |

## 6. API 设计

### 6.1 提交反馈

```
POST /api/feedback/submit
```

**请求参数:**

```json
{
  "type": "function",
  "description": "导航时地图突然变成空白",
  "images": ["base64_image_1", "base64_image_2"],
  "attach_log": true,
  "device_info": {
    "platform": "Android",
    "os_version": "13",
    "device_model": "Xiaomi 13",
    "app_version": "1.0.0+build140"
  },
  "user_info": {
    "user_id": "u_abc123"
  },
  "log_snippet": "..."
}
```

**响应:**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "feedback_id": "fb_20240319143025001"
  }
}
```

### 6.2 获取反馈列表（管理后台）

```
GET /api/feedback/list?page=1&size=20
```

**响应:**

```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": "fb_20240319143025001",
        "type": "function",
        "description": "导航时地图突然变成空白",
        "status": "pending",
        "created_at": "2026-03-19T14:30:25Z"
      }
    ],
    "total": 100
  }
}
```

## 7. 埋点需求
