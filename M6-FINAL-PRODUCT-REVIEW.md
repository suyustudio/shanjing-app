## 五、业务流程验证

### 5.1 评论发布流程 ✅

```
用户点击写评价
    ↓
填写评分(必填) + 内容(可选) + 标签(可选) + 照片(可选)
    ↓
前端验证（评分必填、字数限制、照片数量）
    ↓
POST /v1/trails/{trailId}/reviews
    ↓
后端：验证用户、创建评论、更新路线统计
    ↓
返回成功 + 更新评论列表
```

**验证结果**: 流程完整，符合 PRD 2.2

### 5.2 照片上传流程 ✅

```
用户选择照片（最多9张）
    ↓
前端请求上传凭证 POST /v1/photos/upload
    ↓
后端返回 OSS 直传凭证
    ↓
前端直传 OSS
    ↓
上传完成创建记录 POST /v1/photos
    ↓
关联路线/位置信息
```

**验证结果**: 符合 PRD 3.2 OSS 直传设计

### 5.3 关注/粉丝流程 ✅

```
用户点击关注按钮
    ↓
POST /v1/users/{userId}/follow (Toggle)
    ↓
后端：检查是否已关注
    ↓
已关注 → 取消关注 → 更新统计
未关注 → 创建关注 → 更新统计
    ↓
返回 isFollowing 状态
    ↓
前端更新 UI + 显示动画
```

**验证结果**: 符合 PRD 4.2

### 5.4 收藏夹管理流程 ✅

```
用户点击收藏
    ↓
显示收藏夹选择弹窗
    ↓
选择/创建收藏夹
    ↓
POST /v1/collections/{id}/trails
    ↓
收藏成功，显示反馈
```

**验证结果**: 符合 PRD 5.3
