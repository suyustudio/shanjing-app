## 1. 测试计划概述

### 1.1 测试目标

本测试计划旨在确保山径APP M5阶段新增功能的质量，包括：
- **新手引导功能**: 首次用户体验优化
- **成就系统**: 用户激励与留存提升
- **推荐算法**: 个性化路线推荐

### 1.2 测试范围

| 模块 | 优先级 | 功能点 | 测试类型 |
|------|--------|--------|----------|
| 新手引导 | P1 | 4步引导流程、权限申请、完成率 | 功能、流程、埋点 |
| 成就系统 | P2 | 5类成就、4级徽章、分享功能 | 功能、数据、边界 |
| 推荐算法 | P2 | 5因子排序、多样性、性能 | 算法、性能、兼容性 |
| 回归测试 | P0 | M4功能完整性、数据兼容性 | 回归、兼容、升级 |

### 1.3 里程碑计划

```
Week 1 (03/20-03/26): 新手引导测试
├── Day 1-2: 测试环境准备、测试用例评审
├── Day 3-4: 功能测试执行
└── Day 5-6: 完成率测试、问题修复验证

Week 2 (03/27-04/02): 成就系统测试
├── Day 1-2: 成就触发条件测试
├── Day 3-4: 徽章升级逻辑测试
└── Day 5-6: 本地数据持久化、分享功能测试

Week 3 (04/03-04/09): 推荐算法测试
├── Day 1-2: 5因子排序正确性测试
├── Day 3-4: 推荐结果多样性测试
└── Day 5-6: 性能测试、冷启动测试

Week 4 (04/10-04/16): 回归测试 & 验收
├── Day 1-3: M4功能回归测试
├── Day 4-5: 兼容性测试、升级流程测试
└── Day 6: 验收测试、报告产出
```

## 2. 测试策略

### 2.1 测试类型分布

| 测试类型 | 占比 | 主要内容 |
|----------|------|----------|
| 功能测试 | 40% | 核心业务流程验证 |
| 性能测试 | 20% | 推荐算法性能、动画流畅度 |
| 兼容性测试 | 15% | 多设备、多系统版本 |
| 回归测试 | 15% | M4功能完整性 |
| 数据测试 | 10% | 持久化、兼容性、迁移 |

### 2.2 测试优先级

| 优先级 | 定义 | 通过标准 |
|--------|------|----------|
| **P0 - 阻塞** | 核心功能不可用 | 100% 通过 |
| **P1 - 重要** | 主要功能有缺陷 | 严重问题修复后可延期 |
| **P2 - 一般** | 次要功能问题 | 记录并安排后续修复 |

### 2.3 测试准入标准

```
开发交付检查项:
□ 代码审查通过
□ 单元测试覆盖率 > 80%
□ 静态代码检查无严重问题
□ 自测报告已提交
□ 测试数据已准备
```

## 3. P1 功能测试: 新手引导

### 3.1 测试目标

验证新手引导功能在各种场景下的正确性，确保新用户能够顺利完成引导流程。

### 3.2 测试重点

| 测试项 | 测试内容 | 关键指标 |
|--------|----------|----------|
| 首次启动流程 | 首次安装后自动触发 | 100%触发 |
| 跳过功能 | 可随时跳过引导 | 响应 < 1s |
| 重新触发 | 设置页可重置引导 | 重置后立即生效 |
| 权限申请 | 位置、存储、通知权限 | 不阻塞流程 |
| 4步完成率 | 从欢迎到场景化引导完成 | > 80% |

### 3.3 测试场景矩阵

| 场景 | 触发条件 | 预期结果 |
|------|----------|----------|
| 全新安装 | 首次启动APP | 显示欢迎页 |
| 升级安装 | 从M4升级 | 不显示引导 |
| 重置数据 | 清除数据后启动 | 显示引导 |
| 跳过引导 | 任意步骤点击跳过 | 进入首页 |
| 权限拒绝 | 用户拒绝权限 | 友好提示，可继续 |
| 后台恢复 | 引导中切后台 | 恢复后保持当前页 |

## 4. P2 功能测试: 成就系统

### 4.1 测试目标

验证5类成就的触发条件和4级徽章升级逻辑，确保成就系统准确激励用户行为。

### 4.2 成就类型测试

| 成就类型 | 测试重点 | 测试数据 |
|----------|----------|----------|
| **探索类** | 路线完成计数 | 模拟完成1/5/15/30/50条路线 |
| **里程类** | 累计距离计算 | 模拟10/50/100/500公里里程 |
| **频率类** | 连续徒步周数 | 模拟2/4/8/16周连续 |
| **挑战类** | 特殊条件触发 | 夜间徒步、雨天徒步、独自徒步 |
| **社交类** | 分享/互动计数 | 分享10/30/100次 |

### 4.3 徽章升级测试

```
升级路径验证:
探索类 - 路线收集家
├── 铜(5条) → 银(15条): 增量10条触发升级
├── 银(15条) → 金(30条): 增量15条触发升级
├── 金(30条) → 钻石(50条): 增量20条触发升级
└── 降级保护: 数据回滚不会降级徽章

里程类 - 行者无疆
├── 铜(10km) → 银(50km)
├── 银(50km) → 金(100km)
└── 金(100km) → 钻石(500km)
```

### 4.4 边界测试

| 边界场景 | 测试数据 | 预期结果 |
|----------|----------|----------|
| 临界值触发 | 刚好达到升级条件 | 正确触发升级 |
| 并发解锁 | 多个成就同时满足 | 按序显示动画 |
| 离线解锁 | 无网络时达成条件 | 本地记录，联网同步 |
| 数据清除 | 清除APP数据后 | 成就数据从服务端恢复 |
| 时间边界 | 周成就跨周计算 | 按自然周计算 |

## 5. P2 功能测试: 推荐算法

### 5.1 测试目标

验证推荐算法的正确性、多样性和性能，确保用户获得高质量的个性化推荐。

### 5.2 5因子排序测试

| 因子 | 权重 | 测试方法 | 验证点 |
|------|------|----------|--------|
| 地理位置 | 30% | 不同位置测试 | 距离近的优先 |
| 难度匹配 | 25% | 模拟不同用户 | 符合用户能力 |
| 距离偏好 | 20% | 改变历史数据 | 推荐距离匹配 |
| 评分 | 15% | 控制评分变量 | 高分优先 |
| 新鲜度 | 10% | 新路线测试 | 新路线适当加权 |

### 5.3 多样性测试

| 测试场景 | 测试方法 | 通过标准 |
|----------|----------|----------|
| 类型多样性 | 检查推荐结果 | 不同类型路线混排 |
| 难度多样性 | 模拟新用户 | 覆盖各难度等级 |
| 地理多样性 | 不同城市测试 | 不局限单一区域 |
| 冷启动 | 全新用户 | 有默认推荐列表 |

### 5.4 性能测试指标

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 推荐计算时间 | < 100ms | 服务端API响应时间 |
| 列表加载时间 | < 1s | 端到端时间 |
| 并发推荐 | 支持1000 QPS | 压力测试 |
| 内存占用 | < 50MB | 推荐列表内存 |

## 10. 风险与应对

### 10.1 已知风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 推荐算法数据不足 | 高 | 推荐效果差 | 准备模拟数据，降低预期 |
| 成就触发条件复杂 | 中 | 边界情况遗漏 | 增加边界测试用例 |
| 多设备兼容问题 | 中 | 功能异常 | 提前进行兼容性测试 |
| 性能不达标 | 低 | 体验下降 | 预留优化时间 |

### 10.2 应急预案

| 场景 | 应急措施 |
|------|----------|
| 测试进度延迟 | 增加测试人员，聚焦P0功能 |
| 阻塞问题过多 | 召开专项会议，Dev/QA联合攻关 |
| 测试环境故障 | 切换到备用环境，使用本地数据 |
