# P10 边界场景：权限拒绝

## 场景：用户拒绝定位权限

### 触发条件
用户首次打开地图功能时，系统弹出定位权限请求，用户点击"拒绝"。

### 用户提示文案

**弹窗标题：** 需要定位权限

**正文：**
> 为了为您推荐附近的餐厅和计算徒步距离，我们需要访问您的位置信息。

**按钮：**
- 【暂不开启】—— 关闭弹窗，使用默认位置
- 【去开启】—— 跳转系统设置

---

### 引导开启权限的交互

```
┌─────────────────────────────┐
│                             │
│     📍 需要定位权限          │
│                             │
│  为了为您推荐附近的餐厅      │
│  和计算配送距离，我们需      │
│  要访问您的位置信息。        │
│                             │
│   ┌─────────┐ ┌─────────┐   │
│   │暂不开启 │ │ 去开启  │   │
│   └─────────┘ └─────────┘   │
│                             │
└─────────────────────────────┘
```

#### 交互流程

1. **用户点击【暂不开启】**
   - 弹窗关闭
   - 页面显示默认位置（如：北京市中心）
   - 顶部显示提示条："手动选择位置 >"

2. **用户点击【去开启】**
   - 跳转系统设置页（iOS: 设置 > 隐私 > 定位服务）
   - 用户返回App后，自动重新请求定位

#### 降级方案
- 显示城市选择入口
- 支持手动输入地址搜索
- 地图默认定位到城市中心

---

## 备注
- 仅首次拒绝时弹出引导，避免打扰
- 后续可在"我的 > 设置 > 隐私"中重新开启
