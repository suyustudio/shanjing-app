## 1. 测试概述

### 1.1 测试目标

通过真实徒步场景验证山径APP的核心功能，确保产品在真实环境中的可用性和稳定性。

| 目标维度 | 具体目标 | 成功标准 |
|---------|---------|---------|
| **功能验证** | GPS定位、导航、离线地图、SOS | 功能可用率 100% |
| **数据准确性** | 路线数据、坐标精度 | 坐标偏差 < 20米 |
| **用户体验** | 操作流程、界面响应 | 无阻塞性问题 |
| **稳定性** | 长时间使用无异常 | 30分钟无崩溃 |

### 1.2 测试路线信息

| 属性 | 详情 |
|------|------|
| **路线名称** | 西湖环湖线 |
| **路线ID** | R009 |
| **起点** | 断桥残雪 |
| **终点** | 断桥残雪（环线） |
| **总距离** | 约 10.8 公里 |
| **预计耗时** | 2.5 - 3 小时 |
| **难度等级** | 休闲（Easy） |
| **路况特点** | 平路为主，部分石板路、木栈道 |

### 1.3 路线途经点

```
路线轨迹:
断桥残雪 → 白堤 → 平湖秋月 → 苏堤春晓 → 花港观鱼 → 
雷峰塔 → 柳浪闻莺 → 音乐喷泉 → 断桥残雪（终点）

关键坐标点:
1. 断桥残雪: 30.2596, 120.1479
2. 白堤: 30.2588, 120.1432
3. 苏堤春晓: 30.2315, 120.1289
4. 雷峰塔: 30.2311, 120.1498
5. 柳浪闻莺: 30.2420, 120.1556
```

### 3.5 测试阶段五：SOS压力测试 ⭐新增

**⚠️ 重要提示：此阶段需在安全可控环境下进行，确保不会触发真实求救**

| 步骤 | 操作 | 预期结果 | 实测结果 |
|------|------|---------|---------|

| 5.2 | **弱网环境测试**：开启2G网络模拟，触发SOS并等待倒计时结束 | 请求超时后提示"网络较弱，已本地保存，信号恢复后自动发送" | |
| 5.3 | **无信号场景测试**：开启飞行模式，触发SOS | 提示"无网络信号，求救信息已保存，请尽快移动到信号区域" | |
| 5.4 | **后台触发测试**：将APP切换到后台，等待5分钟后点击SOS悬浮窗（如有） | APP能正常唤醒并进入SOS倒计时 | |
| 5.5 | **电量极低测试**：电量降至5%以下时触发SOS | SOS功能正常可用，其他非核心功能已降级 | |
| 5.6 | **中断恢复测试**：SOS发送过程中断网，恢复网络后检查自动重发 | 信号恢复后自动补发求救信息 | |
| 5.7 | **埋点验证**：检查所有SOS相关事件的埋点上报 | sos_trigger/sos_cancel/sos_failed等事件正常上报 | |
