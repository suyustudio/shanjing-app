### 🟠 P1 问题（强烈建议修复）

#### P1-1: 评分计算算法与 PRD 不符
**描述**: PRD 2.5 定义了复杂的评分计算规则：
- 去掉最高最低各5%后取平均
- 最近30天评价权重×1.2
- 认证用户标识权重

**当前实现**: 简单算术平均 `ratingSum / totalCount`

**代码位置**: `reviews.service.ts:350-360`

**建议实现**:
```typescript
private async calculateWeightedRating(reviews: Review[]): Promise<number> {
  if (reviews.length === 0) return 0;
  
  // 去掉最高最低各5%
  const sorted = [...reviews].sort((a, b) => Number(a.rating) - Number(b.rating));
  const trimCount = Math.ceil(sorted.length * 0.05);
  const trimmed = sorted.slice(trimCount, sorted.length - trimCount);
  
  // 加权计算（最近30天权重×1.2）
  const thirtyDaysAgo = new Date(Date.now() - 30 * 24 * 60 * 60 * 1000);
  let weightedSum = 0;
  let totalWeight = 0;
  
  for (const review of trimmed) {
    const isRecent = review.createdAt > thirtyDaysAgo;
    const weight = isRecent ? 1.2 : 1.0;
    weightedSum += Number(review.rating) * weight;
    totalWeight += weight;
  }
  
  return Number((weightedSum / totalWeight).toFixed(1));
}
```

#### P1-2: "体验过"认证标识缺失
**描述**: PRD 2.5 提到"完成过该路线的用户标识'体验过'"，Schema 和 API 均未实现。

**影响**: 用户无法区分真实体验者和未体验者，降低评论可信度。

**建议**:
1. 在 `reviews` 表中添加 `isVerified` 布尔字段（Schema 已有，但未使用）
2. 在 `CreateReviewDto` 中添加 `isVerified` 字段
3. 在 `ReviewDto` 中返回 `isVerified`
4. 验证逻辑：检查用户是否完成过该路线（查询用户徒步记录）

#### P1-3: 评论图片编辑功能缺失
**描述**: `UpdateReviewDto` 中没有 `photos` 字段，用户无法在编辑时修改图片。

**当前行为**: 编辑评论时只能修改评分、内容、标签，不能修改图片。

**建议**: 在 `UpdateReviewDto` 中添加可选的 `photos` 字段，并在 `updateReview` 方法中处理图片的增删。
