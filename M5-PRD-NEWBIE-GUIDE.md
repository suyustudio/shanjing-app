## 一、功能概述

### 1.1 功能名称
新手引导 (New User Onboarding)

### 1.2 功能定位
在用户首次启动应用时，通过沉浸式引导流程帮助用户快速了解产品核心价值，获取必要权限，建立使用信心。

### 1.3 目标用户
- 首次下载并打开应用的新用户
- 未完整完成引导流程的用户

### 1.4 参考竞品
**AllTrails 首次启动体验:**
- 精美的自然风光图片作为背景
- 简洁有力的价值主张文案
- 渐进式权限申请说明
- 流畅的滑动切换动效

---

## 二、用户故事

### 2.1 故事 1：初次探索
> 作为新用户，我希望在第一次打开应用时，能快速了解山径 App 的核心价值，这样我就知道这款 App 能帮我做什么。

**验收标准:**
- 引导页清晰展示产品核心价值
- 每页停留时间控制在 3-5 秒阅读时长
- 支持左右滑动浏览

### 2.2 故事 2：权限授权
> 作为新用户，我希望了解为什么需要位置、存储等权限，这样我才愿意授权。

**验收标准:**
- 在权限说明页解释每项权限的用途
- 权限申请时机合理，不突兀
- 说明文案通俗易懂

### 2.3 故事 3：自主控制
> 作为用户，我希望可以随时跳过引导，之后还能重新查看，这样我不会感到被强制。

**验收标准:**
- 每页都有"跳过"按钮
- 设置中提供"重新查看新手引导"入口
- 跳过状态不影响其他功能使用

## 三、功能流程

### 3.1 流程图

```
首次启动 App
    │
    ▼
[Step 1: 欢迎页]
    │ (展示品牌价值 + 精美户外图片)
    ▼
[Step 2: 权限说明]
    │ (位置/存储/通知权限说明)
    ▼
[Step 3: 核心功能]
    │ (发现路线/开始徒步/记录轨迹)
    ▼
[Step 4: 完成引导]
    │ (欢迎语 + 开始使用按钮)
    ▼
进入首页
```

### 3.2 分支流程

```
任意步骤点击"跳过"
    │
    ▼
确认弹窗"确定要跳过引导吗？"
    │
    ├── 确定 → 进入首页
    └── 取消 → 返回当前引导页
```

## 五、交互规范

### 5.1 导航规则
| 操作 | 响应 |
|------|------|
| 右滑 | 进入下一页（最后一页时无响应） |
| 左滑 | 返回上一页（第一页时无响应） |
| 点击指示器 | 跳转到对应页面 |
| 点击跳过 | 弹出确认对话框 |
| 点击开始探索 | 进入首页并标记完成 |

### 5.2 动画规范
| 动画 | 时长 | 缓动 |
|------|------|------|
| 页面切换 | 300ms | ease-in-out |
| 指示器切换 | 200ms | ease-out |
| 按钮出现 | 400ms | ease-out |
| 完成页庆祝动效 | 1000ms | spring |

### 5.3 跳过确认弹窗
```
标题: 确定跳过？
内容: 你可以在设置中随时重新查看新手引导
按钮: [取消] [确定跳过]
```

## 六、数据埋点

### 6.1 事件列表

| 事件名 | 触发时机 | 参数 |
|--------|----------|------|
| onboarding_start | 首次进入引导 | - |
| onboarding_page_view | 每页展示 | page_index: int |
| onboarding_next | 点击下一步/右滑 | from_page: int, to_page: int |
| onboarding_skip_click | 点击跳过按钮 | current_page: int |
| onboarding_skip_confirm | 确认跳过 | current_page: int |
| onboarding_complete | 完成全部引导 | duration_seconds: int |
| onboarding_permission_grant | 授权成功 | permission_type: string |
| onboarding_permission_deny | 授权拒绝 | permission_type: string |

### 6.2 关键指标
- 引导完成率: 目标 > 70%
- 每页停留时长: 平均 3-5 秒
- 跳过率: 目标 < 30%
- 权限授权率: 目标 > 80%

## 七、异常处理

### 7.1 权限被拒绝
- 在权限说明页标注"可在设置中开启"
- 不阻断引导流程，允许继续
- 在首页功能使用时再提示授权

### 7.2 中途退出 App
- 记录当前引导进度
- 下次启动时恢复到上次位置
- 提供"从头开始"选项

### 7.3 网络异常
- 引导流程不依赖网络
- 所有资源本地预置

## 八、技术实现要点

### 8.1 存储逻辑
```dart
// 引导完成标记
SharedPreferences.setBool('onboarding_completed', true);

// 当前进度
SharedPreferences.setInt('onboarding_current_page', index);
```

### 8.2 入口判断
```dart
bool shouldShowOnboarding() {
  final completed = prefs.getBool('onboarding_completed') ?? false;
  return !completed;
}
```

### 8.3 重新触发
- 设置页面增加"重新查看新手引导"入口
- 点击后重置 `onboarding_completed` 为 false
