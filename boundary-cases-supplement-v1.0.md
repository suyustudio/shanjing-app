## 1. 新增边界场景

### 1.1 电量相关场景

#### 场景：电量低警告

**触发条件**
- 设备电量低于 20%
- 导航进行中

**用户提示文案**
```
标题: 电量不足
正文: 当前电量 18%，建议尽快结束导航或连接充电器
按钮: [我知道了] [结束导航]
```

**自动处理**
- 弹窗提醒用户
- 自动进入省电模式：
  - 降低GPS采样频率（1秒→5秒）
  - 关闭非必要动画
  - 降低屏幕亮度建议

---

#### 场景：电量极低（<10%）

**触发条件**
- 设备电量低于 10%
- 导航进行中

**用户提示文案**
```
标题: 电量极低
正文: 当前电量 8%，建议立即保存进度并结束导航
按钮: [保存并结束] [继续导航]
```

**自动处理**
- 自动保存当前导航进度
- 发送位置给紧急联系人（如已设置）
- 强烈建议用户结束导航

---

### 1.2 存储空间相关场景

#### 场景：存储空间不足

**触发条件**
- 下载离线包前检查存储空间
- 可用空间 < 100MB

**用户提示文案**
```
标题: 存储空间不足
正文: 可用空间 85MB，需要 150MB 下载该离线包
      建议清理旧离线包或其他应用数据
按钮: [清理空间] [取消]
```

**处理流程**
1. 提示用户清理空间
2. 提供快捷入口到离线包管理页面
3. 用户清理后重新检查空间

---

#### 场景：存储空间严重不足

**触发条件**
- 可用空间 < 50MB

**用户提示文案**
```
标题: 存储空间严重不足
正文: 可用空间不足，无法下载离线包
      请清理手机存储后重试
按钮: [去清理] [取消]
```

**降级方案**
- 阻止下载操作
- 引导用户使用在线导航
- 提供存储清理建议

---

### 1.3 异常退出与恢复

#### 场景：APP异常退出

**触发条件**
- APP崩溃或被系统杀死
- 导航进行中

**自动保存**
- 每30秒自动保存导航状态
- 保存内容：
  - 当前位置
  - 已走路程
  - 开始时间
  - 路线ID

---

#### 场景：恢复导航

**触发条件**
- APP重新启动
- 检测到未完成的导航记录

**用户提示文案**
```
标题: 恢复导航？
正文: 检测到您在 九溪十八涧 的导航尚未完成
      已走 3.2km，用时 45分钟
按钮: [继续导航] [结束导航] [放弃记录]
```

**处理流程**
```
APP启动
    ↓
检查未完成的导航记录
    ↓
有未完成记录 ──→ 显示恢复弹窗
    ↓
├─ 继续导航 ──→ 恢复到导航页面，继续记录
├─ 结束导航 ──→ 显示导航总结，保存记录
└─ 放弃记录 ──→ 删除临时记录，进入首页
```

---

### 1.4 离线包过期

#### 场景：离线包过期提醒

**触发条件**
- 启动APP时检查离线包有效期
- 离线包超过 30 天未更新

**用户提示文案**
```
标题: 离线地图需要更新
正文: 您的离线地图数据已过期（35天前下载）
      建议更新以获取最新地图数据
按钮: [立即更新] [稍后提醒] [忽略]
```

**处理流程**
- 在离线包管理页面标记过期状态
- 导航前提示更新（如使用过期离线包）
- 自动更新检查（WiFi环境下）

---

### 1.5 导航过程打断

#### 场景：导航中来电话

**触发条件**
- 导航进行中
- 系统来电

**处理逻辑**
- 自动暂停导航记录（但不结束）
- 通话结束后提示恢复导航
- 语音播报暂停

---

#### 场景：导航中切换APP

**触发条件**
- 导航进行中
- 用户切换到其他APP

**处理逻辑**
- 后台继续导航（需后台定位权限）
- 保持语音播报
- 显示系统通知栏导航状态

**通知栏显示**
```
🧭 山径 - 导航中
九溪十八涧 | 已走 2.5km | 剩余 2.3km
[停止导航]
```

---

### 1.6 地图数据异常

#### 场景：地图数据加载失败

**触发条件**
- 离线地图文件损坏
- 地图数据解析错误

**用户提示文案**
```
标题: 地图数据异常
正文: 离线地图数据加载失败，建议重新下载
错误码: MAP_DATA_001
按钮: [重新下载] [使用在线地图] [取消]
```

**处理流程**
1. 尝试使用在线地图（如有网络）
2. 无网络时提示重新下载
3. 记录错误日志上报

---

#### 场景：轨迹数据异常

**触发条件**
- 路线轨迹数据解析失败
- 轨迹坐标点异常

**降级方案**
- 显示路线起点和终点
- 隐藏轨迹线
- 提示用户联系客服

```dart
// lib/utils/battery_monitor.dart

import 'package:battery_plus/battery_plus.dart';

class BatteryMonitor {
  static final Battery _battery = Battery();
  static StreamSubscription? _subscription;
  
  static void startMonitoring() {
    _subscription = _battery.onBatteryStateChanged.listen((state) async {
      final level = await _battery.batteryLevel;
      
      if (level <= 20 && level > 10) {
        _showLowBatteryWarning(level);
      } else if (level <= 10) {
        _showCriticalBatteryWarning(level);
      }
    });
  }
  
  static void _showLowBatteryWarning(int level) {
    // 显示低电量警告，进入省电模式
  }
  
  static void _showCriticalBatteryWarning(int level) {
    // 显示严重警告，建议结束导航
  }
}
```

### 4.2 导航状态保存

```dart
// lib/services/navigation_storage.dart

class NavigationStorage {
  static const String _key = 'unfinished_navigation';
  
  static Future<void> saveNavigationState(NavigationState state) async {
    final prefs = await SharedPreferences.getInstance();
    await prefs.setString(_key, jsonEncode(state.toJson()));
  }
  
  static Future<NavigationState?> getUnfinishedNavigation() async {
    final prefs = await SharedPreferences.getInstance();
    final data = prefs.getString(_key);
    if (data != null) {
      return NavigationState.fromJson(jsonDecode(data));
    }
    return null;
  }
  
  static Future<void> clearNavigationState() async {
    final prefs = await SharedPreferences.getInstance();
    await prefs.remove(_key);
  }
}
```
